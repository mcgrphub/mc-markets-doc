The Resources endpoints provide full CRUD access to your data resources. Use them to list and filter resources, create new records with optional metadata, retrieve individual items, update resource properties, and permanently delete records you no longer need.

---

## List resources

<br />

**`GET /v1/resources`**

Returns a paginated list of resources. You can filter by status or owner to narrow your results.

### Query parameters

<ParamField query="page" type="integer" default={1}>
  The page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer" default={20}>
  The number of resources to return per page. Defaults to `20`. Maximum is `100`.
</ParamField>

<ParamField query="status" type="string">
  Filter resources by status. One of `active` or `archived`.
</ParamField>

<ParamField query="owner_id" type="string">
  Filter resources by the ID of the owning user. Returns only resources belonging to the specified user.
</ParamField>

### Example request

```bash
curl -X GET "https://api.example.com/v1/resources?page=1&per_page=20&status=active&owner_id=usr_01HXYZ1234ABCDEF" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "data": [
    {
      "id": "res_01HXYZ9999MNOPQR",
      "name": "Production Dataset",
      "description": "Primary dataset for the production pipeline.",
      "status": "active",
      "owner_id": "usr_01HXYZ1234ABCDEF",
      "metadata": {
        "region": "us-east-1",
        "tier": "premium"
      },
      "created_at": "2024-02-20T08:00:00Z",
      "updated_at": "2024-06-15T10:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 1,
    "total_pages": 1
  }
}
```

### Response fields — resource object

<ResponseField name="id" type="string">
  The unique identifier for the resource.
</ResponseField>

<ResponseField name="name" type="string">
  The display name of the resource.
</ResponseField>

<ResponseField name="description" type="string">
  An optional human-readable description of the resource.
</ResponseField>

<ResponseField name="status" type="string">
  The current status of the resource. One of `active` or `archived`.
</ResponseField>

<ResponseField name="owner_id" type="string">
  The unique identifier of the user who owns this resource.
</ResponseField>

<ResponseField name="metadata" type="object">
  An optional free-form object for storing custom key-value pairs associated with the resource.
</ResponseField>

<ResponseField name="created_at" type="string">
  The date and time the resource was created, formatted as an ISO 8601 timestamp.
</ResponseField>

<ResponseField name="updated_at" type="string">
  The date and time the resource was last updated, formatted as an ISO 8601 timestamp.
</ResponseField>

## Create a resource

<br />

**`POST /v1/resources`**

Creates a new resource. Returns the newly created resource object with a `201 Created` status.

### Request body

<ParamField body="name" type="string" required>
  The display name of the resource.
</ParamField>

<ParamField body="description" type="string">
  A human-readable description of the resource.
</ParamField>

<ParamField body="status" type="string" default="active">
  The initial status of the resource. One of `active` or `archived`. Defaults to `active`.
</ParamField>

<ParamField body="metadata" type="object">
  An optional free-form object for storing custom key-value pairs. All values must be strings or numbers.
</ParamField>

### Example request

```bash
curl -X POST "https://api.example.com/v1/resources" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Staging Dataset",
    "description": "Dataset used for staging environment tests.",
    "status": "active",
    "metadata": {
      "region": "eu-west-1",
      "tier": "standard"
    }
  }'
```

### Example response

```json
{
  "id": "res_02HABC1111STUVWX",
  "name": "Staging Dataset",
  "description": "Dataset used for staging environment tests.",
  "status": "active",
  "owner_id": "usr_01HXYZ1234ABCDEF",
  "metadata": {
    "region": "eu-west-1",
    "tier": "standard"
  },
  "created_at": "2024-07-11T09:00:00Z",
  "updated_at": "2024-07-11T09:00:00Z"
}
```

<Note>
  A successful creation returns HTTP `201 Created`. The `owner_id` is automatically set to the authenticated user associated with your API key.
</Note>

## Get a resource

<br />

**`GET /v1/resources/{id}`**

Retrieves a single resource by its unique ID. Returns the resource object with a `200 OK` status.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the resource to retrieve.
</ParamField>

### Example request

```bash
curl -X GET "https://api.example.com/v1/resources/res_02HABC1111STUVWX" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "id": "res_02HABC1111STUVWX",
  "name": "Staging Dataset",
  "description": "Dataset used for staging environment tests.",
  "status": "active",
  "owner_id": "usr_01HXYZ1234ABCDEF",
  "metadata": {
    "region": "eu-west-1",
    "tier": "standard"
  },
  "created_at": "2024-07-11T09:00:00Z",
  "updated_at": "2024-07-11T09:00:00Z"
}
```

### Error — resource not found

If no resource with the given `id` exists, the API returns a `404 Not Found` response:

```json
{
  "error": {
    "code": "not_found",
    "message": "No resource found with id 'res_02HABC1111STUVWX'."
  }
}
```

## Update a resource

<br />

**`PUT /v1/resources/{id}`**

Updates an existing resource. Include only the fields you want to change. Returns the updated resource object with a `200 OK` status.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the resource to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  The updated display name of the resource.
</ParamField>

<ParamField body="description" type="string">
  The updated description.
</ParamField>

<ParamField body="status" type="string">
  The updated status. One of `active` or `archived`.
</ParamField>

<ParamField body="metadata" type="object">
  The updated metadata object. This **replaces** the existing metadata entirely — include all key-value pairs you want to retain.
</ParamField>

### Example request

```bash
curl -X PUT "https://api.example.com/v1/resources/res_02HABC1111STUVWX" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Staging Dataset v2",
    "status": "archived",
    "metadata": {
      "region": "eu-west-1",
      "tier": "standard",
      "archived_by": "alice"
    }
  }'
```

### Example response

```json
{
  "id": "res_02HABC1111STUVWX",
  "name": "Staging Dataset v2",
  "description": "Dataset used for staging environment tests.",
  "status": "archived",
  "owner_id": "usr_01HXYZ1234ABCDEF",
  "metadata": {
    "region": "eu-west-1",
    "tier": "standard",
    "archived_by": "alice"
  },
  "created_at": "2024-07-11T09:00:00Z",
  "updated_at": "2024-09-01T13:15:00Z"
}
```

<Tip>
  To archive a resource without deleting it, send a `PUT` request with `"status": "archived"`. Archived resources remain retrievable but are excluded from the default list response unless you filter by `status=archived`.
</Tip>

## Delete a resource

<br />

**`DELETE /v1/resources/{id}`**

Permanently deletes a resource. Returns `204 No Content` on success with an empty response body.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the resource to delete.
</ParamField>

### Example request

```bash
curl -X DELETE "https://api.example.com/v1/resources/res_02HABC1111STUVWX" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
  **Deletion is permanent and cannot be undone.** All data associated with the resource, including its metadata and history, is immediately removed. If you need a reversible option, archive the resource by setting `status` to `archived` instead.
</Warning>
