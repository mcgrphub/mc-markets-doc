Endpoints tagged **Deal Order (成交订单)** query user deal (fill) records. All endpoints in this group require the `READ` permission.

## Endpoints

| Method | Path | Permission | Description |
| --- | --- | --- | --- |
| GET | `/openapi/v1/deal-order` | `READ` | Paginated query of deal orders. |
| GET | `/openapi/v1/deal-order/{dealId}` | `READ` | Query details by deal ID. |

---

## Paginated query — deal orders

`GET /openapi/v1/deal-order` — permission `READ`.

### Query parameters

Standard pagination fields: `size`, `current`, `orderBy`, `orderDirection`, etc.

### Response `data` — BasePageRsp\<DealOrderRemarkPageRsp\>

Key list-row fields: `remark`, `dealAmount` / `dealAmountDisplay`, `commission` / `commissionDisplay`, `profit` / `profitDisplay` (and corresponding scale fields).

```bash cURL
curl -s \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/deal-order?current=1&size=20"
```

---

## Deal order details

`GET /openapi/v1/deal-order/{dealId}` — permission `READ`.

### Path parameters

| Parameter | Type | Description |
| --- | --- | --- |
| `dealId` | long | Deal order ID |

### Response `data` — DealOrderDetailRsp

Key fields: `dealId`, `orderId`, `positionId`, `dealTime` / `dealTimeTimestamp`, `type`, `symbolPair`, `entry`, `price`, `pricePosition`, `lot`, `volumeClosed`, `commission`, `profit`, `storage`, `stopLoss`, `takeProfit`, `onChainStatus`, `txHash`, `accountType`, `remark`, etc.

```bash cURL
curl -s \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/deal-order/100234"
```
