Endpoints tagged **Finance Account (理财账户)** query finance products and positions, and manage demand/fixed-term subscriptions and redemptions.

## Endpoints

| Method | Path | Permission | Description |
| --- | --- | --- | --- |
| GET | `/openapi/v1/finance/products/catalog` | `READ` | Query enabled product catalog by type (full list, not paginated). |
| GET | `/openapi/v1/finance/account/positions/holding` | `READ` | Paginated query of current positions (demand positions for the same product are aggregated into one row). |
| GET | `/openapi/v1/finance/account/positions/history` | `READ` | Paginated query of user redemption details. |
| GET | `/openapi/v1/finance/account/overview` | `READ` | C-side finance overview: totals, product list, per-product user balances. |
| POST | `/openapi/v1/finance/account/deposit` | `TRADE` | Subscribe from spot → fund account → finance product. |
| POST | `/openapi/v1/finance/account/withdraw` | `TRADE` | Redeem **demand** finance products back to spot. |
| POST | `/openapi/v1/finance/account/withdraw/fixed` | `TRADE` | Redeem a **fixed-term** position (principal \+ interest). |

<Note>
  Demand and fixed-term redemption permissions are `TRADE` (not `WITHDRAW`).
</Note>

## Product catalog

`GET /openapi/v1/finance/products/catalog` — permission `READ`.

Returns the full list of enabled products by type (`DEMAND` / `FIXED`), sorted by base annual rate \+ bonus annual rate descending.

### Query parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `type` | string | Yes | `DEMAND` (demand) or `FIXED` (fixed-term) |

```bash cURL
curl -s \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/finance/products/catalog?type=DEMAND"
```

### Response `data` — FinanceProductCatalogRsp[]

Key fields: `productCode`, `productName`, `type`, `currency`, `annualRate`, `rewardAnnualRate`, `maxAnnualRate`, `periodDays`, `minDepositAmount`, `minRedeemAmount`, `digit`, `subscribedAmount`, `fundPoolAmount`, `availableSubscriptionBalance`, `subscriptionStartTime` / `subscriptionEndTime` (and millisecond timestamps), `maxDepositAmount`, `dailyMaxRedeemAmount`.

## Current positions

`GET /openapi/v1/finance/account/positions/holding` — permission `READ`.

Paginated query of holding and matured positions. Demand positions for the same product are aggregated into one row (`positionId` may be `null`). List is sorted by `product_code`, then `currency` ascending.

`positionAmount` is the total principal (excluding unsettled interest; see `currentRewardAmount` for interest). `cumulativeRewardAmount` is the historical settled-interest total in the original currency, not converted to USDC.

### Query parameters

Standard pagination fields: `size`, `current`, `orderBy`, `orderDirection`, etc. (see [Pagination](/concepts/pagination)).

```bash cURL
curl -s \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/finance/account/positions/holding?current=1&size=20"
```

## Subscribe to a finance product

`POST /openapi/v1/finance/account/deposit` — permission `TRADE`.

Transfers from the user's spot account into the fund account, then subscribes to the specified finance product.

### Request body — FinanceDepositReq

```bash cURL
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -d '{ "trackingId": "track-dep-0001", "productCode": "USDT_CURRENT", "amount": 100 }' \
  "https://test-gateway.mcconnects.com/openapi/v1/finance/account/deposit"
```

On success, `data` is a transaction ID string.

## Redeem — fixed-term position

`POST /openapi/v1/finance/account/withdraw/fixed` — permission `TRADE`.

Redeems a fixed-term position by `positionId` (holding or matured). The backend redeems principal \+ interest in full.

### Request body — FinanceFixedWithdrawReq

```bash cURL
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \

  "https://test-gateway.mcconnects.com/openapi/v1/finance/account/withdraw/fixed"
```

## Finance overview (C-side)

`GET /openapi/v1/finance/account/overview` — permission `READ`.

Products are returned in `product_code` ASC, then `currency` ASC order. Per-user amounts come from position summaries under each product. `totalDepositValue` is the sum of position principals × oracle price in USDC. `totalInterestValue` and `yesterdayInterestValue` come from interest-settlement records, converted to USDC. `userSpotAvailableBalance` is per-spot-currency and shared across products with the same currency.

```bash cURL
curl -s \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/finance/account/overview"
```
