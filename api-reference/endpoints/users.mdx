The Users endpoints let you manage user accounts in your organization. Use them to list all users, create new accounts, retrieve individual profiles, update user details, and permanently remove users from your system.

---

## List users

<br />

**`GET /v1/users`**

Returns a paginated list of all users in your organization. You can narrow results using the `search` parameter to filter by name or email.

### Query parameters

<ParamField query="page" type="integer" default={1}>
  The page number to retrieve. Defaults to `1`.
</ParamField>

<ParamField query="per_page" type="integer" default={20}>
  The number of users to return per page. Defaults to `20`. Maximum is `100`.
</ParamField>

<ParamField query="search" type="string">
  Filter users by name or email address. Performs a case-insensitive partial match.
</ParamField>

### Example request

```bash
curl -X GET "https://api.example.com/v1/users?page=1&per_page=20&search=alice" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "data": [
    {
      "id": "usr_01HXYZ1234ABCDEF",
      "name": "Alice Nguyen",
      "email": "alice@example.com",
      "status": "active",
      "created_at": "2024-01-15T09:30:00Z",
      "updated_at": "2024-06-01T14:22:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "per_page": 20,
    "total": 1,
    "total_pages": 1
  }
}
```

### Response fields — user object

<ResponseField name="id" type="string">
  The unique identifier for the user.
</ResponseField>

<ResponseField name="name" type="string">
  The user's full name.
</ResponseField>

<ResponseField name="email" type="string">
  The user's email address.
</ResponseField>

<ResponseField name="status" type="string">
  The user's current status. One of `active` or `inactive`.
</ResponseField>

<ResponseField name="created_at" type="string">
  The date and time the user was created, formatted as an ISO 8601 timestamp (e.g. `2024-01-15T09:30:00Z`).
</ResponseField>

<ResponseField name="updated_at" type="string">
  The date and time the user was last updated, formatted as an ISO 8601 timestamp.
</ResponseField>

## Create a user

<br />

**`POST /v1/users`**

Creates a new user account in your organization. Returns the newly created user object with a `201 Created` status.

### Request body

<ParamField body="name" type="string" required>
  The full name of the user.
</ParamField>

<ParamField body="email" type="string" required>
  The email address of the user. Must be unique within your organization.
</ParamField>

<ParamField body="status" type="string" default="active">
  The initial status of the user. One of `active` or `inactive`. Defaults to `active`.
</ParamField>

### Example request

```bash
curl -X POST "https://api.example.com/v1/users" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Bob Martinez",
    "email": "bob@example.com",
    "status": "active"
  }'
```

### Example response

```json
{
  "id": "usr_02HABC5678GHIJKL",
  "name": "Bob Martinez",
  "email": "bob@example.com",
  "status": "active",
  "created_at": "2024-07-10T11:00:00Z",
  "updated_at": "2024-07-10T11:00:00Z"
}
```

<Note>
  A successful creation returns HTTP `201 Created`. The response body contains the complete user object, including the system-assigned `id` and timestamps.
</Note>

## Get a user

<br />

**`GET /v1/users/{id}`**

Retrieves a single user by their unique ID. Returns the user object with a `200 OK` status.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to retrieve.
</ParamField>

### Example request

```bash
curl -X GET "https://api.example.com/v1/users/usr_02HABC5678GHIJKL" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Accept: application/json"
```

### Example response

```json
{
  "id": "usr_02HABC5678GHIJKL",
  "name": "Bob Martinez",
  "email": "bob@example.com",
  "status": "active",
  "created_at": "2024-07-10T11:00:00Z",
  "updated_at": "2024-07-10T11:00:00Z"
}
```

### Error — user not found

If no user with the given `id` exists, the API returns a `404 Not Found` response:

```json
{
  "error": {
    "code": "not_found",
    "message": "No user found with id 'usr_02HABC5678GHIJKL'."
  }
}
```

## Update a user

<br />

**`PUT /v1/users/{id}`**

Updates an existing user's details. You only need to include the fields you want to change — any fields you omit remain unchanged. Returns the updated user object with a `200 OK` status.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to update.
</ParamField>

### Request body

<ParamField body="name" type="string">
  The updated full name of the user.
</ParamField>

<ParamField body="email" type="string">
  The updated email address. Must remain unique within your organization.
</ParamField>

<ParamField body="status" type="string">
  The updated status. One of `active` or `inactive`.
</ParamField>

### Example request

```bash
curl -X PUT "https://api.example.com/v1/users/usr_02HABC5678GHIJKL" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Robert Martinez",
    "status": "inactive"
  }'
```

### Example response

```json
{
  "id": "usr_02HABC5678GHIJKL",
  "name": "Robert Martinez",
  "email": "bob@example.com",
  "status": "inactive",
  "created_at": "2024-07-10T11:00:00Z",
  "updated_at": "2024-08-22T16:45:00Z"
}
```

## Delete a user

<br />

**`DELETE /v1/users/{id}`**

Permanently deletes a user from your organization. Returns `204 No Content` on success with an empty response body.

### Path parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to delete.
</ParamField>

### Example request

```bash
curl -X DELETE "https://api.example.com/v1/users/usr_02HABC5678GHIJKL" \
  -H "Authorization: Bearer YOUR_API_KEY"
```

<Warning>
  **Deletion is permanent.** Once you delete a user, the action cannot be undone and all associated data is immediately removed. Deactivate the user by setting `status` to `inactive` if you want a reversible alternative.
</Warning>
