Endpoints tagged **Trading** execute flash conversion (exchange v2) and query account transaction and fund-flow records.

## Endpoints

| Method | Path | Permission | Description |
| --- | --- | --- | --- |
| POST | `/openapi/v1/transaction/exchange/v2` | `TRADE` | Exchange v2 (supports quote sessions). |
| POST | `/openapi/v1/transaction/transaction/record/transactionId/{id}` | `READ` | Query a single transaction by `transactionId`. |
| POST | `/openapi/v1/transaction/transaction/record/page` | `READ` | Paginated query of transaction records. |
| POST | `/openapi/v1/transaction/account/record/page` | `READ` | Paginated query of fund-flow records. |

All endpoints return the standard envelope. Paginated endpoints return `data` shaped as `{ records: [...], total, size, current, pages, hasPrevious, hasNext }`.

## Exchange v2 (flash conversion)

`POST /openapi/v1/transaction/exchange/v2` — permission `TRADE`.

Executes an exchange with quote-session support.

<Warning>
  The endpoint requires completing the 2FA verification flow before calling. Include `verify2FaTokens` in the body when needed, and ensure you hold a valid `quoteSessionId`.
</Warning>

### Request body — ExchangeV2Req

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `trackingId` | string | Yes | Idempotency tracking ID |
| `quoteSessionId` | string | Yes | Quote session ID |
| `symbol` | string | Yes | Source symbol name, e.g. `USDT` |
| `exchangeSymbol` | string | Yes | Target symbol name, e.g. `BTC` |
| `amount` | number | Yes | Exchange amount, up to 8 decimal places |
| `verify2FaTokens` | array | No | List of temporary 2FA verification tokens |

```bash cURL
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -d '{
        "trackingId": "track-ex-0001",
        "quoteSessionId": "qs_abc123",
        "symbol": "USDT",
        "exchangeSymbol": "BTC",
        "amount": 100
      }' \
  "https://test-gateway.mcconnects.com/openapi/v1/transaction/exchange/v2"
```

On success, `data` is a transaction ID string. Use [Exchange Rate](/api-reference/endpoints/exchange-rate) first to obtain quote context.

## Query transaction record by transactionId

<ParamField path="id" type="string" required>
  The `transactionId` of the record to retrieve. Sent as a path parameter.
</ParamField>

**Permission required:** `READ`

```bash cURL
curl -s -X POST \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/transaction/transaction/record/transactionId/TX-20260101-000001"
```

### Response `data` — TransactionRecordRsp

<ResponseField name="transactionType" type="string">
  Enum. One of `RECHARGE`, `WITHDRAWAL`, `TRANSFER_IN`, `TRANSFER_OUT`, `EXCHANGE`, `SAFEBOX_TRANSFER_IN`, `SAFEBOX_TRANSFER_OUT`, `ROLLBACK`, `ADJUSTMENT_ADD`, `ADJUSTMENT_REDUCE`, `MLP_DEPOSIT`, `MLP_WITHDRAWAL`, `TRANSFER_TRADE_TO_SPOT`, `TRANSFER_SPOT_TO_TRADE`, `EARN_DEPOSIT`, `EARN_WITHDRAWAL`.

  One of `FUND_ACCOUNT`, `TRADING_ACCOUNT`, `SAFE_BOX`, `EXTERNAL_ACCOUNT`, `MLP_ACCOUNT`, `SPOT_ACCOUNT`, `EARN_ACCOUNT`.
</ResponseField>

## Paginated query — transaction records

`POST /openapi/v1/transaction/transaction/record/page`

**Permission required:** `READ`

### Request body — TransactionRecordPageReq

```bash cURL
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -d '{
        "trackingId": "track-20260101-0001",
        "size": 10,
        "current": 1,
        "orderBy": "id,createTime",
        "orderDirection": "DESC,ASC",
        "transactionTypes": ["RECHARGE", "WITHDRAWAL"],
        "createTimeStart": "2026-01-01 00:00:00.000",
        "createTimeEnd": "2026-01-31 23:59:59.999"
      }' \
  "https://test-gateway.mcconnects.com/openapi/v1/transaction/transaction/record/page"
```

    "hasPrevious": false,
    "hasNext": true,

      {
        "transactionId": "TX-20260101-000001",

## Paginated query — fund-flow records

`POST /openapi/v1/transaction/account/record/page`

**Permission required:** `READ`

Request body and response are the same shape as the transaction-records pagination above. This endpoint returns fund-flow records from the `ac_account_record` perspective (per-account debit/credit view) rather than the transaction-level view.

```bash cURL
curl -s -X POST \
  -H "Content-Type: application/json" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  -d '{ "trackingId": "track-20260101-0002", "size": 20, "current": 1 }' \
  "https://test-gateway.mcconnects.com/openapi/v1/transaction/account/record/page"
```
