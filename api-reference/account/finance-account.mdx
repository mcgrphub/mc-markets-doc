## Withdraw from Finance account (flexible)

`POST /openapi/v1/mc-account/finance/account/withdraw` — permission `TRADE`.

Flexible products only: pass the product code and withdrawal amount. Funds move from the Finance account to the fund account, then into spot. For multiple fixed-term positions, use the fixed-term position withdrawal endpoint and specify the position detail ID.

### Request body (JSON)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| trackingId | string | Yes | Unique request tracking ID |
| productCode | string | Yes | Flexible finance product code |
| amount | number | Yes | Withdrawal amount |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| data | string | `data` is a scalar of this type |

```bash
curl -sS -X POST "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/account/withdraw" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{
  "example": "see fields above"
}' 
```

## Withdraw fixed-term Finance position

`POST /openapi/v1/mc-account/finance/account/withdraw/fixed` — permission `TRADE`.

Fixed-term products only: specify the position detail ID (active or matured; matches `positionId` from the current positions endpoint). The backend redeems the full principal plus interest for that position. Workflow matches flexible withdrawal.

### Request body (JSON)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| trackingId | string | Yes | Unique request tracking ID |
| positionId | integer | Yes | Fixed-term position primary key ID (redeemable whether active or matured; matches `positionId` returned by the current positions endpoint) |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| data | string | `data` is a scalar of this type |

  -H "X-Nonce: YOUR_NONCE" \

## Finance product catalog

`GET /openapi/v1/mc-account/finance/products/catalog` — permission `READ`.

Returns the full list of enabled product configurations by type (flexible / fixed-term), unpaginated, sorted descending by base APR plus bonus APR. Query parameter `type` is required.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| type | string | Yes | Product type: flexible / fixed-term |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| data | array | `data` is an array of objects; element fields use `[]...` |
| [].productCode | string | Product Code |
| [].productName | string | Product Name |
| [].type | string | Type: Demand/Term |
| [].currency | string | Currency |
| [].annualRate | number | Base APY% |
| [].rewardAnnualRate | number | Excess reward APY% (config tier) |
| [].maxAnnualRate | number | Max APY% = Base + Reward |
| [].periodDays | integer | Term days, 0 for demand products |
| [].minDepositAmount | number | Minimum Deposit Amount |
| [].minRedeemAmount | number | Minimum Redemption Amount |
| [].digit | integer | Currency precision (decimal places) |
| [].subscribedAmount | number | Subscribed amount: net retained subscription principal in the pool (consistent with fund pool cap verification: cumulative deposits - cumulative redemptions + cumulative interest redemptions) |
| [].fundPoolAmount | number | Pool cap: maximum fund pool (ac_finance_product_config.fund_pool_amount; null or ≤0 means no limit) |
| [].availableSubscriptionBalance | number | Available subscription balance: remaining depositable amount up to the pool cap; null when pool has no cap set |
| [].subscriptionStartTime | string | Subscription start time; if both start and end are null, no restriction applies |
| [].subscriptionEndTime | string | Subscription end time; if both start and end are null, no restriction applies |
| [].maxDepositAmount | number | Max subscription amount: per-user max deposit (ac_finance_product_config.max_deposit_amount; null or ≤0 means no limit) |
| [].dailyMaxRedeemAmount | number | Daily Withdrawal Limit |
| [].subscriptionStartTimeTimestamp | integer | Subscription start time millisecond timestamp |
| [].subscriptionEndTimeTimestamp | integer | Subscription end time millisecond timestamp |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/products/catalog" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "type=VALUE"
```

## Current Finance account positions

`GET /openapi/v1/mc-account/finance/account/positions/holding` — permission `READ`.

Paginated query of active and matured positions for the context user ID. Multiple flexible positions for the same product are aggregated into one row. Results are sorted by `product_code` ascending, then `currency` ascending. `positionAmount` is total position principal (excludes unsettled interest; see `currentRewardAmount` for interest). `cumulativeRewardAmount` is the historical settled interest total in the original currency for that product and currency, not converted to USDC. Returns an empty page when not logged in or when the business line is missing.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| size | integer | No |  |
| current | integer | No |  |
| orderBy | string | No | Multi-field sort supported; comma-separated |
| orderDirection | string | No | ASC ascending / DESC descending; comma-separated for multiple fields |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| records | array |  |
| records[].positionId | integer | Position detail ID; null when multiple demand entries of the same product are aggregated |
| records[].productName | string | Product Name |
| records[].productCode | string | Product Code |
| records[].currency | string | Asset Currency |
| records[].digit | integer | Currency precision (decimal places) |
| records[].lockPeriodDays | integer | Lock-up days (term); 0 for demand products |
| records[].type | string | Type: Demand / Term |
| records[].positionAmount | number | Principal of holding orders: for demand products with multiple entries, this is the sum of principal_balance; excludes unsettled interest and frozen amounts (see currentRewardAmount for interest) |
| records[].baseAnnualRate | number | Base annual interest rate % (term/demand single entry takes position snapshot; demand summary takes product config) |
| records[].rewardMinHoldingDays | integer | Must hold continuously for (days) to enjoy the excess tier; may be null when no tier rules exist |
| records[].excessRewardAnnualRate | number | Excess reward APY% (product config) |
| records[].eligibleRewardAmount | number | Order holding amount not less than: the position principal threshold required to enjoy the excess reward tier (product config reward_min_position; may be null when no excess tier exists) |
| records[].currentRewardAmount | number | Current holding interest: total holding interest for this user, product, and currency in ac_user_finance_position. Unit is the product's asset currency (consistent with currency field), not converted to USDC |
| records[].cumulativeRewardAmount | number | Historical total interest: sum of amount from successful INTEREST records in ac_finance_transaction for this user, product, and currency. Unit is the product's asset currency (consistent with currency field), not converted to USDC |
| records[].status | string | Position status: Holding / Matured (for demand product summaries, if any entry is holding, the status is Holding) |
| records[].subscribeTime | string | Term: subscription time for this position detail (ac_user_finance_position.create_time, when subscription was successfully recorded). Null for demand product summaries |
| records[].excessRewardAnnualRateDisplay | string | Excess reward APY display value |
| records[].eligibleRewardAmountDisplay | string | Reward position principal threshold display value |
| records[].currentRewardAmountDisplay | string | Current holding interest display value |
| records[].cumulativeRewardAmountDisplay | string | Historical total interest display value |
| records[].baseAnnualRateDisplay | string | Base annual interest rate display value |
| records[].subscribeTimeTimestamp | integer | Subscription time millisecond timestamp |
| records[].positionAmountDisplay | string | Holding order principal display value |

| size | integer |  |
| current | integer |  |
| pages | integer |  |
| hasPrevious | boolean |  |
| hasNext | boolean |  |

  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \

## User redemption history

`GET /openapi/v1/mc-account/finance/account/positions/history` — permission `READ`.

Aligns with admin redemption history: main table `ac_transaction_history` (`EARN_WITHDRAWAL`), paginated by `create_time`. `productCode`, type, position linkage, etc. come from `ac_finance_transaction` (`tx_id` = `transaction_id`, `WITHDRAW`). Redemption amount and time come from the main ledger entry `amount` and `create_time`.

## Finance account overview

`GET /openapi/v1/mc-account/finance/account/overview` — permission `READ`.

Returns `products` by product dimension, ordered by `product_code` ascending, then `currency` ascending. User-side `userDepositedAmount`, `userInterestAmount`, and `availableBalance` come from position aggregates in `ac_user_finance_position` for that product (in the product currency). `totalDepositValue` is the sum of each product's position principal × oracle price, converted to USDC. `totalInterestValue` and `yesterdayInterestValue` are interest settlement records converted to USDC by currency. `userSpotAvailableBalance` is per trading-account currency; products sharing the same currency have the same value. User-side values are 0 when not logged in.

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| totalDepositValue | number | User total deposit (principal per currency × corresponding oracle price, converted to USDC; 0 when not logged in) |
| totalInterestValue | number | User historical settled interest total (ac_finance_transaction successful INTEREST records aggregated by currency, amount × oracle price, converted to USDC. Calculation consistent with totalDepositValue and yesterdayInterestValue: decimal p |
| yesterdayInterestValue | number | Yesterday's interest (settled interest records from the previous calendar day in Asia/Shanghai timezone, converted to USDC using the same oracle; 0 when not logged in) |
| products | array | By product dimension: each entry corresponds to ac_finance_product + configuration. User amounts come from position summaries under that product, platform fields come from product summary tables. Includes subscribedAmount, fundPoolAmount, a |
| products[].productCode | string | Product Code |
| products[].type | string | Product type: Demand / Term |
| products[].currency | string | Currency |
| products[].lockPeriodDays | integer | Lock-up days (term); 0 for demand products |
| products[].maxDepositAmount | number | Per-user subscription cap (per-user max deposit; null or ≤0 means no limit) |
| products[].dailyMaxRedeemAmount | number | Daily redemption limit (null or ≤0 means no limit) |
| products[].pauseRedeem | boolean | Whether redemption is suspended (true means suspended, redemption cannot be initiated) |
| products[].annualRate | number | Base APY% (excluding point rewards) |
| products[].rewardAnnualRate | number | Reward annual rate: excess reward tier APY% (config excess_reward_annual_rate) |
| products[].totalAnnualRate | number | Total APY% = Base + Reward (for display) |
| products[].minDepositAmount | number | Minimum Deposit Amount |
| products[].minRedeemAmount | number | Minimum Withdrawal Amount |
| products[].subscriptionStartTime | string | Subscription start time; if both start and end are null, no restriction applies |
| products[].subscriptionEndTime | string | Subscription end time; if both start and end are null, no restriction applies |
| products[].rewardMinHoldingDays | integer | Order holding days: must hold continuously for (days) to qualify for the excess reward tier (reward_min_holding_days) |
| products[].rewardMinPosition | number | Order holding amount not less than: must satisfy together with holding days for tiered interest calculation (reward_min_position) |
| products[].oraclePrice | number | Oracle price (quote currency) |
| products[].userDepositedAmount | number | Total position principal for the current user under this product (ac_user_finance_position.principal_balance, holding + matured; 0 when not logged in) |
| products[].userInterestAmount | number | Total unsettled interest for the current user under this product (ac_user_finance_position.interest_balance; 0 when not logged in) |
| products[].availableBalance | number | Total book value for the current user under this product: principal + interest + frozen (ac_user_finance_position; 0 when not logged in) |
| products[].userSpotAvailableBalance | number | Spot available balance for the current user in this currency (independent of product; same value returned for all products of the same currency; 0 when not logged in) |
| products[].totalDeposited | number | Platform's cumulative deposits for this product (ac_finance_product.total_deposited) |
| products[].totalPrincipal | number | Platform's current retained principal for this product (ac_finance_product: total_deposited - total_withdrawn + total_interest_withdrawn) |
| products[].subscribedAmount | number | Subscribed amount: net retained subscription principal in the pool (consistent with fund pool cap verification: cumulative deposits - cumulative redemptions + cumulative interest redemptions) |
| products[].fundPoolAmount | number | Pool cap: maximum fund pool (ac_finance_product_config.fund_pool_amount; null or ≤0 means no limit) |
| products[].availableSubscriptionBalance | number | Available subscription balance: remaining depositable amount up to the pool cap; null when pool has no cap set |

| products[].oraclePriceDisplay | string | Oracle price display value |
| products[].availableBalanceDisplay | string | User position book value total display value |
