## Withdraw from Finance account (flexible)

`POST /openapi/v1/mc-account/finance/account/withdraw` â€” permission `TRADE`.

Flexible products only: pass the product code and withdrawal amount. Funds move from the Finance account to the fund account, then into spot. For multiple fixed-term positions, use the fixed-term position withdrawal endpoint and specify the position detail ID.

### Request body (JSON)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| trackingId | string | Yes | Unique request tracking ID |
| productCode | string | Yes | Flexible finance product code |
| amount | number | Yes | Withdrawal amount |

```bash
curl -sS -X POST "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/account/withdraw" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE" \
  -H "Content-Type: application/json" \
  -d '{
  "example": "see fields above"
}' 
```

## Withdraw fixed-term Finance position

`POST /openapi/v1/mc-account/finance/account/withdraw/fixed` â€” permission `TRADE`.

Fixed-term products only: specify the position detail ID (active or matured; matches `positionId` from the current positions endpoint). The backend redeems the full principal plus interest for that position. Workflow matches flexible withdrawal.

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| trackingId | string | Yes | Unique request tracking ID |
| positionId | integer | Yes | Fixed-term position primary key ID (redeemable whether active or matured; matches `positionId` returned by the current positions endpoint) |


## Finance product catalog

`GET /openapi/v1/mc-account/finance/products/catalog` â€” permission `READ`.

Returns the full list of enabled product configurations by type (flexible / fixed-term), unpaginated, sorted descending by base APR plus bonus APR. Query parameter `type` is required.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| type | string | Yes | Product type: flexible / fixed-term |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/products/catalog" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "type=VALUE"
```

## Current Finance account positions

`GET /openapi/v1/mc-account/finance/account/positions/holding` â€” permission `READ`.

Paginated query of active and matured positions for the context user ID. Multiple flexible positions for the same product are aggregated into one row. Results are sorted by `product_code` ascending, then `currency` ascending. `positionAmount` is total position principal (excludes unsettled interest; see `currentRewardAmount` for interest). `cumulativeRewardAmount` is the historical settled interest total in the original currency for that product and currency, not converted to USDC. Returns an empty page when not logged in or when the business line is missing.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| size | integer | No |  |
| current | integer | No |  |
| orderBy | string | No | Multi-field sort supported; comma-separated |
| orderDirection | string | No | ASC ascending / DESC descending; comma-separated for multiple fields |

### Response `data` fields (summary)

| Field | Type | Description |
| --- | --- | --- |
| records | array |  |
| total | integer |  |
| size | integer |  |
| current | integer |  |
| pages | integer |  |
| hasPrevious | boolean |  |
| hasNext | boolean |  |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/account/positions/holding" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
```

## Finance account overview

`GET /openapi/v1/mc-account/finance/account/overview` â€” permission `READ`.

Returns `products` by product dimension, ordered by `product_code` ascending, then `currency` ascending. User-side `userDepositedAmount`, `userInterestAmount`, and `availableBalance` come from position aggregates in `ac_user_finance_position` for that product (in the product currency). `totalDepositValue` is the sum of each product's position principal Ă— oracle price, converted to USDC. `totalInterestValue` and `yesterdayInterestValue` are interest settlement records converted to USDC by currency. `userSpotAvailableBalance` is per trading-account currency; products sharing the same currency have the same value. User-side values are 0 when not logged in.

### Response `data` fields (summary)

| Field | Type | Description |
| --- | --- | --- |
| totalDepositValue | number | Total user deposits (principal per currency Ă— corresponding oracle price, converted to USDC; 0 when not logged in) |
| totalInterestValue | number | Total historical settled interest (successful `INTEREST` records in `ac_finance_transaction`, summed by currency, then Ă— same overview oracle prices, converted to USDC; calculation basis matches `totalDepositValue` and `yesterdayInterestValue`) |
| yesterdayInterestValue | number | Yesterday's interest (settled interest records credited on the previous calendar day in Asia/Shanghai, converted to USDC using the same oracle prices; 0 when not logged in) |
| products | array | By product dimension: each row maps to `ac_finance_product` + config; user amounts from position aggregates under that product, platform fields from product summary table; includes `subscribedAmount`, `fundPoolAmount`, `availableSubscriptionBal` |
| totalDepositValueDisplay | string | Total user deposits display value |
| totalInterestValueDisplay | string | Total historical settled interest display value |
| yesterdayInterestValueDisplay | string | Yesterday's interest display value |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-account/finance/account/overview" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
```
