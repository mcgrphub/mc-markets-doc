## Endpoints

| Method | Path | Permission | Description |
| --- | --- | --- | --- |
| GET | /openapi/v1/mc-trade/account/positions | READ | Fund movement list |
| GET | /openapi/v1/mc-trade/account/positions/current/v2 | READ | Query current positions V2 |
| GET | /openapi/v1/mc-trade/account/pnl | READ | PnL statistics for a time range |
| GET | /openapi/v1/mc-trade/account/contract-volume-by-symbol | Public | Query total long/short contract volume by trading symbol |
| GET | /openapi/v1/mc-trade/account/contract-notional-by-symbol | Public | Query total long/short contract notional by trading symbol |

<Note>

</Note>

## Fund movement list

`GET /openapi/v1/mc-trade/account/positions` — permission `READ`.

Query the trading account fund movement list (collateral increase/decrease history) for the current user. UID is taken from the `X-UID` request header.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| size | integer | No | Page size |
| current | integer | No | Current page number |
| orderBy | string | No | Sort fields, comma-separated |
| sortOrder | string | No | Sort direction: ASC / DESC |
| login | string | Yes | MT5 contract account ID / spot account ID |
| from | integer | Yes | Start timestamp (Unix seconds) |
| to | integer | Yes | End timestamp (Unix seconds) |
| symbol | string | No | Trading symbol name (optional filter) |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| records | array | Records on the current page |
| total | integer | Total number of records |
| size | integer | Page size |
| current | integer | Current page number |
| pages | integer | Total number of pages |
| hasPrevious | boolean | Whether a previous page exists |
| hasNext | boolean | Whether a next page exists |

`records[]` items:

| Field | Type | Description |
| --- | --- | --- |
| order | integer | Order number, primary key |
| login | integer | Login account, MT5 account login number |
| dealer | integer | Dealer ID, the MT5 dealer code |
| symbol | string | Trading symbol, e.g., EURUSD, GBPUSD, etc. |
| digits | integer | Price precision, the number of decimal places for the trading symbol |
| digitsCurrency | integer | Currency precision, the number of currency decimal places for the trading symbol |
| contractSize | number | Contract size, the standard size per lot |
| state | integer | Order Status |
| reason | integer | Reason, the reason code for the order change |
| timeSetup | integer | Order time, the timestamp when the order was created, in milliseconds, 13 digits |
| timeExpiration | integer | Expiry time, the expiration timestamp of the order, in milliseconds, 13 digits |
| timeDone | integer | Completion time, the timestamp when the order was completed, in milliseconds, 13 digits |
| timeSetupMsc | integer | Order time (ms), the timestamp when the order was created (milliseconds), 13 digits |
| timeDoneMsc | integer | Completion time (ms), the timestamp when the order was completed (milliseconds), 13 digits |
| modifyFlags | integer | Modification flags, the flag bits for order modification |
| type | integer | Order type, e.g., buy, sell, etc. |
| typeFill | integer | Deal type, the fill type of the order |
| typeTime | integer | Time validity type, the validity type of the order |
| priceOrder | number | Order price, the execution price of the order |
| priceOrderStr | string | Order price raw string |
| priceOrderDisplay | string | Order price display value |
| priceOrderScale | integer | Order price display decimal places |
| priceTrigger | number | Trigger price, the trigger price of the order |
| priceTriggerDisplay | string | Trigger price display value |
| priceTriggerScale | integer | Trigger price display decimal places |
| priceCurrent | number | Current price, the current market price of the order |
| priceCurrentDisplay | string | Current price display value |
| priceCurrentScale | integer | Current price display decimal places |
| priceSlDisplay | string | Stop loss price display value |
| priceSlScale | integer | Stop loss price display decimal places |
| priceTpDisplay | string | Take profit price display value |
| priceTpScale | integer | Take profit price display decimal places |
| volumeInitialExt | number | Initial extended volume, the initial extended volume of the order (for large orders) |
| quantity | number | User quantity, equals initial volumeInitialExt divided by 10^8 |
| volumeInitialDisplay | string | Initial volume display value |
| volumeInitialScale | integer | Initial volume display decimal places |
| volumeCurrentExt | number | Current extended volume, the current extended volume of the order |
| volumeCurrentDisplay | string | Current volume display value |
| volumeCurrentScale | integer | Current volume display decimal places |
| rateMargin | number | Margin exchange rate, the exchange rate used when calculating margin |
| activationMode | integer | Activation mode, the activation mode of the pending order |
| activationTime | integer | Activation time, the activation timestamp of the pending order |
| activationPrice | number | Activation price, the activation price of the pending order |
| activationFlags | integer | Activation flag, the flag bit for pending order activation |
| volumeInitial | number | Initial volume, the initial trading volume of the order |
| volumeCurrent | number | Current volume, the current trading volume of the order |
| externalID | string | External ID, the ID associated with the external system |
| priceSL | number | Stop loss price, the set stop loss level |
| priceTP | number | Take profit price, the set take profit level |
| expertID | integer | Expert Advisor ID. If the order was created by an EA (Expert Advisor), records the EA ID |
| positionID | integer | Position ID, the associated position ID |
| positionByID | integer | Opposite position ID, the associated opposite position ID |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/positions" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "size=1" \
  --data-urlencode "current=1" \
  --data-urlencode "login=VALUE" \
  --data-urlencode "from=1755792000"
```

## Query current positions V2

`GET /openapi/v1/mc-trade/account/positions/current/v2` — permission `READ`.

Query all open positions for the current user (UID from the `X-UID` header). MT5 often stores floats as integers to avoid precision issues: fields without an Ext suffix use 4 decimal places (divide by 10000); fields with an Ext suffix use 8 decimal places (divide by 100000000). For example, `mt5_positions` stores volume as `Volume` and `VolumeExt`.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| login | string | Yes | MT5 contract account ID / spot account ID |
| symbol | string | No | Trading symbol |
| positionType | string | No | Omit or pass ALL for all, COLLATERAL for collateral only, SYMBOL for trading positions only |

### Response `data` fields

`data` is an array of objects. Element fields follow.

`data[]` items:

| Field | Type | Description |
| --- | --- | --- |
| position | integer | Position number, the unique position identifier in the MT5 system |
| login | integer | Login account, MT5 account login number |
| dealer | integer | Dealer ID, the MT5 dealer code |
| symbol | string | Trading symbol, e.g., EURUSD, GBPUSD, etc. |
| action | integer | Operation type: 0-Buy, 1-Sell |
| digits | integer | Price precision, the number of decimal places for the trading symbol |
| digitsCurrency | integer | Currency precision, the number of currency decimal places for the trading symbol |
| profitCurrencyDigits | integer | P/L currency display precision. Priority: position DigitsCurrency > mt5_symbols.CurrencyProfitDigits > fallback 2 |
| reason | integer | Reason, the reason code for the position change |
| contractSize | number | Contract size, the standard size per lot |
| timeCreate | integer | Creation time, the timestamp when the position was created, in milliseconds, 13 digits |
| timeUpdate | integer | Update time, the timestamp when the position was last updated, in milliseconds, 13 digits |
| timeCreateMsc | integer | Creation time (ms), the timestamp when the position was created (milliseconds), 13 digits |
| timeUpdateMsc | integer | Update time (ms), the timestamp when the position was last updated (milliseconds), 13 digits |
| priceOpen | number | Open price, the opening price of the position |
| priceOpenDisplay | string | Open price display value |
| priceOpenScale | integer | Open price display decimal places |
| priceCurrent | number | Current price, the current market price of the position |
| priceCurrentDisplay | string | Current price display value |
| priceCurrentScale | integer | Current price display decimal places |
| priceSlDisplay | string | Stop loss price display value |
| priceSlScale | integer | Stop loss price display decimal places |
| priceTpDisplay | string | Take profit price display value |
| priceTpScale | integer | Take profit price display decimal places |
| volumeExt | integer | Extended volume, the extended volume of the position (for large orders). Divide by 100000000 to convert to a floating point number |
| volumeDisplay | string | Position volume display value |
| volumeScale | integer | Position volume display decimal places |
| positionValueDisplay | string | Position value display value |
| positionValueScale | integer | Position value display decimal places |
| profit | number | Profit/Loss, the floating profit/loss amount of the current position |
| profitDisplay | string | Profit/Loss display value |
| profitScale | integer | Profit/Loss display decimal places |
| storage | number | Storage fee, the overnight interest/storage fee of the position |
| rateProfit | number | Profit/Loss exchange rate, the exchange rate used when calculating P/L |
| rateMargin | number | Margin exchange rate, the exchange rate used when calculating margin |
| activationMode | integer | Activation mode, the activation mode of the pending order |
| activationTime | integer | Activation time, the activation timestamp of the pending order |
| activationPrice | number | Activation price, the activation price of the pending order |
| activationFlags | integer | Activation flag, the flag bit for pending order activation |
| volume | integer | Volume, the trading volume of the position in lots. Divide by 10000 to convert to a floating point number |
| fundingRate | number | Funding Fee |
| interest | number | Interest |
| externalID | string | External ID, the ID associated with the external system |
| priceSL | number | Stop loss price, the set stop loss level |
| priceTP | number | Take profit price, the set take profit level |
| expertID | integer | Expert Advisor ID. If the position was created by an EA (Expert Advisor), records the EA ID |
| expertPositionID | integer | Expert Advisor position ID, the position ID associated with EA-created positions |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/positions/current/v2" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "login=VALUE"
```

## PnL statistics for a time range

`GET /openapi/v1/mc-trade/account/pnl` — permission `READ`.

Query PnL for a time range (for example, the past week). UID is taken from the `X-UID` header. If `from` and `to` are omitted, defaults to the last seven days.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| from | integer | No | Start timestamp (seconds), optional |
| to | integer | No | End timestamp (seconds), optional |
| login | string | Yes | Trading account login |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| uid | integer | Account Login Number |
| totalProfit | number | Total Period Profit |
| totalLoss | number | Total Period Loss (negative value) |
| netPnl | number | Net Period P/L |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/pnl" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "login=VALUE"
```

## Query total long/short contract volume by trading symbol

`GET /openapi/v1/mc-trade/account/contract-volume-by-symbol` — permission `Public`.

`symbol` maps to `mt5_positions.Symbol`; only symbols whose Path starts with CFD are counted; volume = VolumeExt/1e8.

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| symbol | string | Yes | Trading symbol |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| symbol | string | Trading symbol |
| longVolume | number | Total long position volume |
| shortVolume | number | Total short position volume |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/contract-volume-by-symbol" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "symbol=BTCUSDT"
```

## Query total long/short contract notional by trading symbol

`GET /openapi/v1/mc-trade/account/contract-notional-by-symbol` — permission `Public`.

`symbol` maps to `mt5_positions.Symbol`; only symbols whose Path starts with CFD are counted; notional = VolumeExt/1e8 * PriceCurrent * RateProfit * ContractSize (not differentiated by Symbol).

### Query / path parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| symbol | string | Yes | Trading symbol |

### Response `data` fields

| Field | Type | Description |
| --- | --- | --- |
| symbol | string | Trading symbol |
| longNotional | number | Total long notional value (2 decimals, truncated down) |
| shortNotional | number | Total short notional value (2 decimals, truncated down) |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/contract-notional-by-symbol" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "symbol=BTCUSDT"
```
