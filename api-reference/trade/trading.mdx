Paths in this group use the `/openapi/v1/mc-trade/` prefix.

## Endpoints

| Method | Path | Permission | Description |
| --- | --- | --- | --- |
| PUT | /openapi/v1/mc-trade/trade/positions/modify | TRADE | Modify position |
| PUT | /openapi/v1/mc-trade/trade/orders/modify | TRADE | Order action: modify order |
| POST | /openapi/v1/mc-trade/trade/orders/create | TRADE | Order action: place order |
| POST | /openapi/v1/mc-trade/trade/orders/close | TRADE | Order action: close position |
| GET | /openapi/v1/mc-trade/trade/symbol/staticConfig | Public | Trading symbol info |
| GET | /openapi/v1/mc-trade/trade/slippage/estimate | READ | Get estimated slippage |
| GET | /openapi/v1/mc-trade/trade/mt5-group/staticInfo | READ | MT5 group static info |
| GET | /openapi/v1/mc-trade/trade/account/assets | READ | Get user assets |
| DELETE | /openapi/v1/mc-trade/trade/orders/cancel | TRADE | Order action: cancel order |

<Note>
  The interactive Playground is disabled. Sign requests with AK/SK as described in [Authentication](/authentication). See [Base URL](/api-reference/base-url) for the gateway host.
</Note>

## Modify position

`PUT /openapi/v1/mc-trade/trade/positions/modify` — permission `TRADE`.

### Request body (JSON)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| login | string | Yes | MT5 contract account ID / spot account ID |
| order | integer | Yes | Order ID |
| symbol | string | Yes | Trading pair, e.g. BTCUSDT; valid only when placing orders |
| type | integer | Yes | Order type: "0-OP_BUY: market buy", "1-OP_SELL: market sell", "2-OP_BUY_LIMIT: limit buy pending order", "3-OP_SELL_LIMIT: limit sell pending order", "4-OP_BUY_STOP: stop buy pending order", "5-OP_SELL_STOP: stop sell pending order", "6-OP_BUY_STOP_LIMIT: stop-limit buy pending order"… |

## Order action: place order

`POST /openapi/v1/mc-trade/trade/orders/create` — permission `TRADE`.

### Request body (JSON)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| login | string | Yes | MT5 contract account ID / spot account ID |
| symbol | string | Yes | Trading pair, e.g. BTCUSDT |
| volumeExt | integer | Yes | Volume scaled by 10^8; for 1 lot pass 1*100000000 |
| price | number | No | Order price; defaults to 0 if omitted; 0 = market, any other value = limit |
| type | integer | Yes | Order type: "0-OP_BUY: market buy", "1-OP_SELL: market sell", "2-OP_BUY_LIMIT: limit buy pending order", "3-OP_SELL_LIMIT: limit sell pending order", "4-OP_BUY_STOP: stop buy pending order", "5-OP_SELL_STOP: stop sell pending order", "6-OP_BUY_STOP_LIMIT: stop-limit buy pending order"… |
| position | integer | No | Position ID; defaults to 0 |
| typeFill | integer | Yes | Fill type: "0-Fill or Kill: the order must be fully filled or cancelled; partial fills are not kept.", "1-Immediate or Cancel: the order may be partially filled; the unfilled remainder is cancelled.", "2-Return remainder to queue: for pending orders (e.g. limit, stop); only for exchange-executed market orders; unfilled volume re-enters the queue.", "3-BOC (Book or Cancel): placed on the book only; if it would execute immediately, it is cancelled." |
| typeTime | integer | No | Time-in-force type; defaults to 0 if omitted. Values: "0-good till cancelled", "1-day order", "2-specified time", "3-specified date; expires at 00:00 on that date or the nearest trading time" |
| expiration | integer | No | Expiration timestamp (required when typeTime=2); defaults to 0 if omitted (no auto-expiry) |
| priceTrigger | number | No | Stop-limit order trigger price; defaults to 0 if omitted |
| priceSL | number | No | Stop-loss |
| priceTP | number | No | Take-profit |
| userSlippageLimit | number | No | Maximum slippage limit for market orders entered by the user (market orders only) |
| frontMarketPrice | number | No | Client-side market reference price (optional); must be ≥ 0 |
