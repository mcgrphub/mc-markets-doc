## API at a glance

| Property | Value |
| --- | --- |
| Product | MC-account (Multi-Account System) |
| Version | `v0.0.1` |
| Base URL (test) | `https://test-gateway.mcconnects.com` |
| Base URL (production) | `https://gateway.mcconnects.com` |
| Path prefix | `/openapi/v1` |
| Protocol | HTTPS only |
| Format | JSON |
| Auth | AK/SK \+ HMAC-SHA256 signature |
| Required headers | `X-Access-Key`, `X-Signature`, `X-Timestamp` |
| Permissions | `READ`, `TRADE`, `WITHDRAW` |

## Response envelope

All endpoints return a common JSON envelope:

```json
{
  "status": 200,
  "code": "200",
  "codeKey": "DEFAULT",
  "message": "...",
  "success": true,
  "data": { }
}
```

- `status` — HTTP status echoed in the body.
- `code` / `codeKey` — business-level status. See the OpenAPI `codeKey` enum for the full list (e.g. `PARAM_ERROR`, `SIGN_ERROR`, `PERMISSION_DENIED`, `ACCOUNT_BALANCE_INSUFFICIENT`).
- `success` — `true` for successful business responses.
- `data` — endpoint-specific payload (object, array, string, or number).

## Common request conventions

- **`trackingId`** — most POST bodies require a `trackingId` string. Use a unique value per request for idempotency and troubleshooting.
- **Pagination request** — page queries share `{ size, current, orderBy, orderDirection, trackingId, ...filters }`. `orderBy` supports comma-separated multi-field sort (e.g. `id,createTime`); `orderDirection` is `ASC` / `DESC`, comma-separated to match.
- **Pagination response** — page responses share `{ records: [...], total, size, current, pages, hasPrevious, hasNext }`.
- **Date-time** — `yyyy-MM-dd HH:mm:ss.SSS` (ISO 8601).
