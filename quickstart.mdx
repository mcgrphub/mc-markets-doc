This quickstart walks you through your first MC OpenAPI call: create an API Key, sign the request with **HMAC-SHA256 signature v2**, and call the exchange-rate endpoint through the gateway.

<Note>
  You need an MC platform account before you begin. If you don't have one, contact your MC representative. From v3.7.0, clients must use signature **v2** (a unique `X-Nonce` on every request).
</Note>

<Steps>
  <Step title="Create an API Key">
    Log in to the MC platform, open **API Key Management**, and click **Create**. Give the key a descriptive name and select the permissions you need:

    - `READ` — read-only operations (balances, lists, queries)
    - `TRADE` — trading operations (place / cancel orders)
    - `WITHDRAW` — withdrawal and transfer operations

    Optionally configure an IP whitelist (up to 15 IPv4 addresses). Copy the **Access Key** and **Secret Key** immediately — the Secret Key is shown only once.

    <Warning>
      Never commit the Secret Key to source control, and never expose it in client-side code.
    </Warning>
  </Step>
  <Step title="Sign a request (v2)">
    Every authenticated request needs at least four headers:

    | Header | Value |
    | --- | --- |
    | `X-Access-Key` | Your Access Key |
    | `X-Timestamp` | Current Unix time in **milliseconds** (within ±10 minutes of server time) |
    | `X-Nonce` | **Unique per request** (UUID v4 recommended) |
    | `X-Signature` | **Base64**-encoded HMAC-SHA256 of the string to sign |

    Build the string to sign by joining the following parts with `\n`, then append a trailing `\n`:

    ```text
    {HTTP_METHOD}
    {REQUEST_URI}
    {QUERY_STRING}       # omit entire line if no query (do not insert a blank line)
    {SIGNED_HEADERS}     # x-access-key, x-nonce, x-timestamp — lowercase keys, sorted
    {TIMESTAMP}
    {NONCE}
    [{BODY_SHA256_HEX}]  # only for POST/PUT/PATCH with a non-empty body
    ```

    See [Authentication](/authentication) for the full rules and Python / Java / Node.js samples.
  </Step>
  <Step title="Call the exchange-rate endpoint">
    `GET /openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT` requires only `READ` and is ideal for verifying your signing implementation.

    ```bash
    ACCESS_KEY='YOUR_ACCESS_KEY'
    SECRET_KEY='YOUR_SECRET_KEY'
    URI='/openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT'
    TIMESTAMP=$(python3 -c 'import time; print(int(time.time()*1000))')
    NONCE=$(python3 -c 'import uuid; print(uuid.uuid4())')
    
    SIGNATURE=$(ACCESS_KEY="$ACCESS_KEY" SECRET_KEY="$SECRET_KEY" URI="$URI" TIMESTAMP="$TIMESTAMP" NONCE="$NONCE" python3 << 'PY'
    import base64, hashlib, hmac, os
    method = "GET"
    uri = os.environ["URI"]
    access_key = os.environ["ACCESS_KEY"]
    secret_key = os.environ["SECRET_KEY"]
    timestamp = os.environ["TIMESTAMP"]
    nonce = os.environ["NONCE"]
    headers = "&".join(
        f"{k}={v}"
        for k, v in sorted(
            {
                "x-access-key": access_key,
                "x-nonce": nonce,
                "x-timestamp": timestamp,
            }.items()
        )
    )
    string_to_sign = f"{method}\n{uri}\n{headers}\n{timestamp}\n{nonce}\n"
    sig = base64.b64encode(
        hmac.new(secret_key.encode(), string_to_sign.encode(), hashlib.sha256).digest()
    ).decode()
    print(sig)
    PY
    )
    
    curl -s \
      -H "X-Access-Key: ${ACCESS_KEY}" \
      -H "X-Signature: ${SIGNATURE}" \
      -H "X-Timestamp: ${TIMESTAMP}" \
      -H "X-Nonce: ${NONCE}" \
      -H "X-Signature-Version: 2" \
      "https://test-gateway.mcconnects.com${URI}"
    ```

    A successful response:

    ```json
    {
      "status": 200,
      "code": "200",
      "success": true,
      "data": 98500.12
    }
    ```
  </Step>
