This quickstart walks you through everything you need to get up and running with the Open API Platform: obtaining an API key and making your first authenticated REST call. By the end, you'll have a working request and a clear path to explore the rest of the platform.

<Note>
  You need a dashboard account before you begin. If you don't have one yet, sign up for free at [https://dashboard.example.com](https://dashboard.example.com).
</Note>

<Steps>
  <Step title="Create an account">
    Navigate to [https://dashboard.example.com](https://dashboard.example.com) and click **Sign Up**. Fill in your name, email address, and a secure password, then verify your email to activate your account.
  </Step>

  <Step title="Generate an API key">
    Once you're logged in, go to **Settings → API Keys** and click **New Key**. Give your key a descriptive name (for example, `my-first-key`) so you can identify it later, then click **Create**.

    Copy your new API key and store it somewhere safe — a password manager or a secrets vault works well.

    <Warning>
      Your API key is only shown once. If you lose it, you'll need to revoke it and generate a new one. Never commit keys to source control.
    </Warning>
  </Step>

  <Step title="Make your first request">
    Use the API key you just generated to call the `/v1/users` endpoint. Replace `YOUR_API_KEY` with your actual key:

    ```bash
    curl https://api.example.com/v1/users \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Accept: application/json"
    ```

    A successful response looks like this:

    ```json
    {
      "data": [
        {
          "id": "usr_01H...",
          "name": "Jane Doe",
          "email": "jane@example.com",
          "created_at": "2024-01-15T10:30:00Z"
        }
      ],
      "pagination": {
        "page": 1,
        "per_page": 20,
        "total": 1
      }
    }
    ```

    The `data` array contains the list of users your key has permission to view. The `pagination` object tells you how many results are available and how they are paged.
  </Step>
