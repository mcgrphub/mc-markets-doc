## API 一览

| 属性 | 值 |
| --- | --- |
| 产品 | MC Markets OpenAPI |
| Base URL(测试) | `https://test-gateway.mcconnects.com` |
| Base URL(生产) | `https://gateway.mcconnects.com` |
| 路径前缀 | `/openapi/v1` |
| 协议 | 仅 HTTPS |
| 格式 | JSON |
| 鉴权 | AK/SK \+ HMAC-SHA256 |
| 必需头 | `X-Access-Key`、`X-Signature`、`X-Timestamp`、`X-Nonce` |
| 权限 | `READ`、`TRADE`、`WITHDRAW` |

## 响应信封

所有接口返回统一的 JSON 信封:

```json
{
  "status": 200,
  "code": "200",
  "codeKey": "DEFAULT",
  "message": "...",
  "success": true,
  "data": { }
}
```

- `status` — body 中回传的 HTTP 状态。
- `code` / `codeKey` — 业务级状态(如 `PARAM_ERROR`、`SIGN_ERROR`、`PERMISSION_DENIED`、`ACCOUNT_BALANCE_INSUFFICIENT`)。
- `success` — 业务成功时为 `true`。
- `data` — 具体接口的数据(对象、数组、字符串或数字)。

## 通用约定

- **`trackingId`** — 大多数 POST body 需传 `trackingId` 字符串,每次请求不同,用于幂等与追踪。
- **分页请求** — 共享 `{ size, current, orderBy, orderDirection, trackingId, ...filters }`。`orderBy` 支持逗号分隔多字段(如 `id,createTime`);`orderDirection` 为 `ASC`/`DESC`,逗号对应。
- **分页响应** — 共享 `{ records: [...], total, size, current, pages, hasPrevious, hasNext }`。
- **时间格式** — `yyyy-MM-dd HH:mm:ss.SSS`(ISO 8601)。
