## 当前持仓查询V2

`GET /openapi/v1/mc-trade/account/positions/current/v2` — 权限 `READ`。

根据当前用户的 UID 查询当前所有未平仓持仓（UID 从请求头 `X-UID` 获取）。MT5 多处用整型表示浮点数以避免精度问题：不带 Ext 后缀的字段表示 4 位小数（转浮点数需除以 10000）；带 Ext 后缀的字段表示 8 位小数（转浮点数需除以 100000000）。例如 `mt5_positions` 表中持仓量有 `Volume` 与 `VolumeExt`。

### 查询 / 路径参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| login | string | 是 | MT5合约账户ID/现货账户ID |
| symbol | string | 否 | 交易品种 |
| positionType | string | 否 | 不传或传ALL-全部，COLLATERAL-仅抵押品，SYMBOL-仅交易持仓 |

### 响应 `data` 字段

`data` 为对象数组。元素字段见下表。

`data[]` 元素：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| position | integer | 持仓编号，持仓唯一标识 |
| login | integer | 交易账户登录号 |
| dealer | integer | 交易商ID |
| symbol | string | 交易品种，如EURUSD、GBPUSD等 |
| action | integer | 操作类型，0-买入，1-卖出 |
| digits | integer | 价格精度，交易品种的价格小数位数 |
| digitsCurrency | integer | 货币精度，交易品种的货币小数位数 |
| profitCurrencyDigits | integer | 盈亏货币展示精度，优先取持仓货币精度，其次品种盈亏货币精度，最后回退 2 |
| reason | integer | 原因，持仓变更的原因代码 |
| contractSize | number | 合约大小，每手合约的标准大小 |
| timeCreate | integer | 创建时间，持仓创建的时间戳，毫秒级，长度13位 |
| timeUpdate | integer | 更新时间，持仓最后更新的时间戳，毫秒级，长度13位 |
| timeCreateMsc | integer | 创建时间（毫秒），持仓创建的时间戳（毫秒），长度13位 |
| timeUpdateMsc | integer | 更新时间（毫秒），持仓最后更新的时间戳（毫秒），长度13位 |
| priceOpen | number | 开仓价格，持仓的开仓价格 |
| priceOpenDisplay | string | 开仓价格展示值 |
| priceOpenScale | integer | 开仓价格展示小数位 |
| priceCurrent | number | 当前价格，持仓的当前市场价格 |
| priceCurrentDisplay | string | 当前价格展示值 |
| priceCurrentScale | integer | 当前价格展示小数位 |
| priceSlDisplay | string | 止损价格展示值 |
| priceSlScale | integer | 止损价格展示小数位 |
| priceTpDisplay | string | 止盈价格展示值 |
| priceTpScale | integer | 止盈价格展示小数位 |
| volumeExt | integer | 持仓量（扩展精度），需按持仓量精度换算 |
| volumeDisplay | string | 持仓手数展示值 |
| volumeScale | integer | 持仓手数展示小数位 |
| positionValueDisplay | string | 持仓价值展示值 |
| positionValueScale | integer | 持仓价值展示小数位 |
| profit | number | 盈亏，当前持仓的浮动盈亏金额 |
| profitDisplay | string | 盈亏展示值 |
| profitScale | integer | 盈亏展示小数位 |
| storage | number | 库存费，持仓的隔夜利息/库存费 |
| rateProfit | number | 盈亏汇率，计算盈亏时使用的汇率 |
| rateMargin | number | 保证金汇率，计算保证金时使用的汇率 |
| activationMode | integer | 激活模式，挂单的激活模式（如挂单激活方式） |
| activationTime | integer | 激活时间，挂单的激活时间戳 |
| activationPrice | number | 激活价格，挂单的激活价格 |
| activationFlags | integer | 激活标志，挂单激活的标志位 |
| volume | integer | 手数（标准精度），需按持仓量精度换算 |
| fundingRate | number | 资金费 |
| interest | number | 利息 |
| externalID | string | 外部ID，外部系统关联的ID |
| priceSL | number | 止损价格，设置的止损价位 |
| priceTP | number | 止盈价格，设置的止盈价位 |
| expertID | integer | 专家顾问ID，如果持仓由 EA 创建，则记录 EA 的 ID |
| expertPositionID | integer | 专家顾问持仓ID，EA创建的持仓关联ID |

  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "login=VALUE"

## 一段时间总盈亏统计

`GET /openapi/v1/mc-trade/account/pnl` — 权限 `READ`。

根据当前用户的 UID 查询一段时间区间内的盈亏统计（如近一周盈亏）。UID 从请求头 `X-UID` 获取；`from` 与 `to` 不传时默认查询最近七天。

### 查询 / 路径参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| from | integer | 否 |  |
| to | integer | 否 |  |
| login | string | 是 |  |

### 响应 `data` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| uid | integer | 账户登录号 |
| totalProfit | number | 区间总盈利 |
| totalLoss | number | 区间总亏损（为负数） |
| netPnl | number | 区间净盈亏 |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/pnl" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "login=VALUE"
```

## 按交易品种 symbol 查询合约多空总持仓名义价值

`GET /openapi/v1/mc-trade/account/contract-notional-by-symbol` — 权限 `公开访问`。

symbol 对应 mt5_positions.Symbol；仅统计 Path 以 CFD 开头的品种；名义 = VolumeExt/1e8 * PriceCurrent * RateProfit * ContractSize（不按 Symbol 区分）。

### 查询 / 路径参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| symbol | string | 是 |  |

### 响应 `data` 字段

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| symbol | string |  |
| longNotional | number |  |
| shortNotional | number |  |

```bash
curl -sS -G "https://gateway.mcconnects.com/openapi/v1/mc-trade/account/contract-notional-by-symbol" \
  -H "X-Access-Key: YOUR_ACCESS_KEY" \
  -H "X-Signature: YOUR_SIGNATURE" \
  -H "X-Timestamp: YOUR_TIMESTAMP" \
  -H "X-Nonce: YOUR_NONCE"
  --data-urlencode "symbol=BTCUSDT"
```
