多数业务响应使用统一 JSON 信封，同时返回数字 `code` 与字符串 `codeKey`。
业务分支请使用 `code` / `codeKey`；`message` / `msg` 仅作可读说明，不要依赖其精确文案。

本页仅收录 **MC OpenAPI 公开接口** 业务调用链中可能返回的错误码。网关 AK/SK 鉴权失败不走此信封，见文末。

```json
{
  "code": 1000005,
  "codeKey": "PARAM_ERROR",
  "msg": "Invalid request parameter",
  "success": false,
  "data": null
}
```

## 编码规则

错误码为 **7 位**：

| 段 | 位数 | 含义 |
| --- | --- | --- |
| `SS` | 2 | 场景 / 平台模块 |
| `BB` | 2 | 业务域 |
| `EEE` | 3 | 具体错误 |

<Note>
  各接口文档不再内联完整 `codeKey` 枚举，请以本页为目录。
</Note>

| `REPEAT_OPERATION` | `1000004` | Repeat operation | `409` |

## 网关鉴权错误（无 `codeKey`）

AK/SK 鉴权由 API 网关处理。失败时通常返回 HTTP **401**（请求体超限为 **413**），响应为：

```json
{
  "status": 401,
  "code": 401,
  "message": "签名无效"
}
```

此类响应**没有**业务信封中的 `codeKey`。常见 `message` 见 [认证](/zh/authentication) 中的「错误响应」。
