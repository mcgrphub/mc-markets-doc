**Funding Fee History（资金费历史记录）** 包含 v2 分页接口,返回按 `positionId + dealTime` 聚合后的资金费与利息费记录。本组接口均需 `READ` 权限。

## 接口

| 方法 | 路径 | 描述 |
| --- | --- | --- |
| GET | `/openapi/v1/funding-fee-record/page/v2` | 分页查询。`feeType` 固定为 `INTEREST_FEE` / `FUNDING_FEE`;记录按 `positionId + dealTime` 聚合,`amount` 为汇总金额。 |

## 分页查询(v2)

`GET /openapi/v1/funding-fee-record/page/v2` — 权限 `READ`。

### Query 参数 — FundingFeeRecordPageReq

`req` 参数为一个对象,HTTP 调用时请将其字段展平为平铺的 query 参数(`?size=10&current=1&...`)。

### 响应 `data` — BasePageRsp\<FundingFeeRecordPageV2Rsp\>

```bash cURL
curl -s \
  -G \
  --data-urlencode "size=20" \
  --data-urlencode "current=1" \
  --data-urlencode "orderBy=id,createTime" \
  --data-urlencode "orderDirection=DESC,ASC" \
  --data-urlencode "dealTimeStart=2026-01-01 00:00:00.000" \
  --data-urlencode "dealTimeEnd=2026-01-31 23:59:59.999" \
  -H "X-Access-Key: ${ACCESS_KEY}" \
  -H "X-Signature: ${SIGNATURE}" \
  -H "X-Timestamp: ${TIMESTAMP}" \
  "https://test-gateway.mcconnects.com/openapi/v1/funding-fee-record/page/v2"
```

<Info>
  构造待签名字符串时,请按 key(ASCII 升序)排序 query 参数,并使用其 URL 编码后的值。
</Info>
