## 环境

| 环境 | Base URL |
| --- | --- |
| 测试 | `https://test-gateway.mcconnects.com` |
| 生产 | `https://gateway.mcconnects.com` |

所有接口共用路径前缀 `/openapi/v1/`。详见 [基础 URL](/zh/api-reference/base-url)。

## 接口分组

MC OpenAPI 接口由 `openapi-exposed` Spec 生成，在 API 参考侧边栏按网关服务分组。

| 分组 | 用途 | 常见权限 |
| --- | --- | --- |
| Account（`/openapi/v1/mc-account/`） | 交易记录、理财、MLP、币种、资金账户、汇率等 | `READ`、`TRADE`、`WITHDRAW` |
| Trade（`/openapi/v1/mc-trade/`） | 订单、持仓、成交、交易账户 | `READ`、`TRADE` |
| Risk（`/openapi/v1/mc-risk/`） | 资金费率、交易品种 | `READ` |
| Aggregator（`/openapi/v1/mc-aggregator/`） | K 线与行情排行 | `READ` |

Spec 文件映射见 [API 参考概览](/zh/api-reference/overview)。

## 请求生命周期

每次调用的流程相同:

1. **构造** 请求—HTTP 方法、路径(`/openapi/v1/...`)、query 与可选 JSON body。
2. **签名** 请求—使用签名 **v2**（含唯一 `X-Nonce`；非空写请求还要纳入 body SHA-256）计算 HMAC-SHA256（Base64）。
3. **发送** —以 HTTPS 发送，带 `X-Access-Key`、`X-Signature`、`X-Timestamp`、`X-Nonce` 头。
4. **处理** 响应—先看 HTTP 状态码,再看 JSON 信封中的 `code`。

<Note>
  时间戳必须位于网关时钟附近。签名时使用当前 UTC 时间的毫秒值。
</Note>
