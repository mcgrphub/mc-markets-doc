## 网关地址

生产 Base URL 为 `https://gateway.mcconnects.com`。所有接口共用路径前缀 `/openapi/v1/`。详见 [基础 URL](/zh/api-reference/base-url)。

## 接口分组

API 参考按业务域分组（文件路径仍按网关服务目录存放）：

| 分组 | 用途 | 常见权限 |
| --- | --- | --- |
| 行情 | 行情数据、币种、汇率 | `READ` |
| 交易 | 下单与持仓、交易账户、订单、成交、账户交易、成交订单、资金费历史、交易品种、资金费率 | `READ`、`TRADE` |
| 理财 | 理财账户、金库账户、资金账户 | `READ`、`TRADE`、`WITHDRAW` |

请求路径仍可能包含 `/openapi/v1/mc-account/`、`/openapi/v1/mc-trade/`、`/openapi/v1/mc-risk/`、`/openapi/v1/mc-aggregator/` 等服务段，签名时须使用完整路径。详见 [API 参考概览](/zh/api-reference/overview)。

## 请求生命周期

每次调用的流程相同:

1. **构造** 请求—HTTP 方法、路径(`/openapi/v1/...`)、query 与可选 JSON body。
2. **签名** 请求—使用新的 `X-Nonce`（**UUID v4**，或 ≥32 位十六进制；非空写请求还要纳入 body SHA-256）计算 HMAC-SHA256（Base64）。
3. **发送** —以 HTTPS 发送，带 `X-Access-Key`、`X-Signature`、`X-Timestamp`、`X-Nonce` 头。
4. **处理** 响应—先看 HTTP 状态码,再看 JSON 信封中的 `code`。

<Note>
  时间戳必须位于网关时钟附近。签名时使用当前 UTC 时间的毫秒值。
</Note>
