MC OpenAPI 是 JSON-over-HTTPS API。每次调用均面向 `/openapi/v1/` 下的网关，携带鉴权所需请求头；请求带 body 时为 JSON。所有响应都是包含 `code`、`msg`、`data`、`success` 的 JSON 信封。

## 请求格式

### 必需头

| Header | 必需 | 值 |
| --- | --- | --- |
| `X-Access-Key` | 是 | 你的 Access Key ID |
| `X-Signature` | 是 | 待签名字符串的 HMAC-SHA256 签名（**Base64**） |
| `X-Timestamp` | 是 | 当前 Unix 时间（**毫秒**） |
| `X-Nonce` | 是 | 每次请求新生成；**UUID v4**，或 ≥32 位十六进制（`0-9a-fA-F`） |
| `Content-Type` | POST/PUT/PATCH 建议 | `application/json` |
| `Accept` | 建议 | `application/json` |
| `X-Signature-Version` | 否 | 建议填 `2`（排障用） |

<Warning>
  缺少有效签名头、`X-Nonce`，或 `X-Timestamp` 超出允许区间的请求将被拒绝。切勿将 Secret Key 提交到代码仓库或在客户端暴露。
</Warning>

待签名字符串的确切格式与 Python/Java/Node.js 参考实现见 [认证](/zh/authentication)。

### 待签名字符串（概要）

```text
{HTTP_METHOD}
{REQUEST_URI}
{QUERY_STRING}
{SIGNED_HEADERS}
{TIMESTAMP}
{NONCE}
[{BODY_SHA256_HEX}]
```

- 无 query 时 **整行省略** `QUERY_STRING`（不要插入空行）。
- `SIGNED_HEADERS` 含 `x-access-key`、`x-nonce`、`x-timestamp`（小写 key，字典序）。
- `POST`/`PUT`/`PATCH` 且 body 非空时，追加 body 原始字节的 SHA-256 **小写 hex**。
- `X-Signature` 为 Base64，不是 hex。

### 示例：查询 USD 汇率

```bash
curl --request GET \
  --url "https://gateway.mcconnects.com/openapi/v1/mc-account/exchange-rate/query/usd" \
  --header "X-Access-Key: $MC_ACCESS_KEY" \
  --header "X-Signature: $MC_SIGNATURE" \
  --header "X-Timestamp: $MC_TIMESTAMP" \
  --header "X-Nonce: $MC_NONCE" \
  --header "Accept: application/json"
```

## 响应格式

所有响应都是 JSON。MC OpenAPI 将每次结果包在统一信封中。

### 响应信封

```json
{
  "code": "0",
  "msg": "success",
  "data": { },
  "success": true
}
```

<ResponseField name="code" type="string">
  业务状态码。`"0"` 代表成功。其他值为业务失败 — 见 [错误处理](/zh/concepts/errors)。
</ResponseField>

<ResponseField name="msg" type="string">
  人类可读的消息。成功时通常为 `"success"`;失败时描述原因。
</ResponseField>

<ResponseField name="data" type="object | array | null">
  实际响应载荷,形式因接口而异。
</ResponseField>

<ResponseField name="success" type="boolean">
  便捷标志;`code == "0"` 时为 `true`。
</ResponseField>

### 分页响应

```json
{
  "code": "0",
  "msg": "success",
  "success": true,
  "data": {
    "pageNum": 1,
    "pageSize": 20,
    "total": 132,
    "pages": 7,
    "list": [
      { "transactionId": "T-1001", "amount": "125.50", "currency": "USDT" }
    ]
  }
}
```

翻页使用方法见 [分页](/zh/concepts/pagination)。

## HTTP 状态码

MC OpenAPI 使用标准 HTTP 状态码表示传输层结果。即使返回 `200 OK`,仍需检查信封中的 `code`。

| 码 | 含义 |
| --- | --- |
| `200` | 请求已送达,检查 JSON 信封中的 `code` 确认业务结果。 |
| `400` | 请求格式错误——参数缺失或 JSON 不合法。 |
| `401` | 签名无效、`X-Timestamp` 偏离或 Access Key 不可识别。 |
| `403` | Access Key 有效但缺少必需权限(`READ`、`TRADE`、`WITHDRAW`)。 |
| `404` | 接口或资源不存在。 |
| `429` | 频率限制——见 [频率限制](/zh/guides/rate-limits)。 |
| `500` | 网关内部错误。指数退避重试。 |
| `503` | 网关临时不可用。指数退避重试。 |

<Tip>
  非 `200` 响应时记录 `code`、`msg` 与响应头中的 correlation ID,便于排障与联系支持。
</Tip>
