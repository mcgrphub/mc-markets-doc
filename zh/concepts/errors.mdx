MC OpenAPI 在**两层**报告失败：标准 HTTP 状态码，以及 JSON 信封中的业务字段（`code` / `codeKey`）。健壮的集成应同时检查两者。

## 两层错误模型

1. **传输层** — HTTP 状态码。`4xx` 为客户端问题，`5xx` 为网关/服务问题。
2. **业务层** — JSON 信封中的 `code` 与 `codeKey`。`200 OK` 仍可能携带业务错误（如余额不足、币种不存在）。

无论 HTTP 状态码如何，都应检查 `success` / `code` / `codeKey`。

## 错误响应格式

失败响应保持相同信封，`success` 为 `false`，并返回非成功业务 `code`：

```json
{
  "code": 1000005,
  "codeKey": "PARAM_ERROR",
  "msg": "Invalid request parameter",
  "success": false,
  "data": null
}
```

<ResponseField name="code" type="integer | string">
  业务状态码。成功时常见为 `0` / `200`（视封装路径而定）。失败时为 **7 位**错误码。
</ResponseField>

<ResponseField name="codeKey" type="string">
  稳定的错误键，例如 `PARAM_ERROR`、`SIGN_ERROR`。分支逻辑优先使用 `codeKey` 或 `code`，不要依赖 `msg`。
</ResponseField>

<ResponseField name="msg" type="string">
  人可读消息，适合日志；不要依赖其精确文案做分支。
</ResponseField>

<ResponseField name="success" type="boolean">
  业务失败时为 `false`。
</ResponseField>

<ResponseField name="data" type="object | null">
  失败时通常为 `null`。部分校验错误会在 `data` 中返回字段级详情。
</ResponseField>

## HTTP 状态码

| 码 | 发生时机 | 应对 |
| --- | --- | --- |
| `400` | JSON 格式错误、参数缺失 | 修正后重试。 |
| `401` | `X-Signature` 缺失/无效或 `X-Timestamp` 偏离 | 校对签名与时钟，重新签名后重试。 |
| `403` | Access Key 缺少必需权限 | 在 Key 管理台勾选 `READ`/`TRADE`/`WITHDRAW`。 |
| `404` | 路径错误或资源 ID 不存在 | 核对 URL 与 ID。 |
| `429` | 频率限制 | 退避重试—见 [频率限制](/zh/guides/rate-limits)。 |
| `500`/`502`/`503` | 网关/上游错误 | 指数退避重试。 |

## 常见业务码

完整目录见 [错误码](/zh/api-reference/error-codes)（仅公开 OpenAPI 可达码）。

| `codeKey` | `code` | 含义 | 建议处理 |
| --- | --- | --- | --- |
| `PARAM_ERROR` | `1000005` | 请求参数无效 | 根据 `msg` 修正参数。 |
| `NOT_EXIST` | `1000002` | 资源不存在 | 核对请求中的 ID。 |
| `ACCOUNT_BALANCE_INSUFFICIENT` | `2004003` | 余额不足 | 充值或减小金额。 |
| `ORDER_OPERATION_FAILED` | `2100001` | 订单操作失败 | 查看 `msg`；仅在瞬时错误时重试。 |
| `DEFAULT` | `1000000` | 内部错误 | 指数退避重试；持续时联系支持。 |

<Note>
  AK/SK 鉴权失败（如 `签名无效`）由网关返回，**没有** `codeKey`，见 [认证](/zh/authentication)。分支逻辑优先使用稳定的 `codeKey`。
</Note>

## 错误处理示例

### Python

```python
import requests, time

def call(method, path, body=""):
    # … 按认证文档构造已签名请求…
    response = requests.request(method, url, headers=headers, data=body)

    if response.status_code == 429 or 500 <= response.status_code < 600:
        raise TransientError(response.status_code, response.text)

    payload = response.json()
    if payload.get("success") is True:
        return payload.get("data")

    code_key = payload.get("codeKey") or str(payload.get("code"))
    if response.status_code == 401:
        raise AuthError(code_key, payload.get("msg") or payload.get("message"))

    raise BusinessError(code_key, payload.get("msg"))


def with_backoff(fn, *args, max_retries=5, **kwargs):
    delay = 1.0
    for attempt in range(max_retries):
        try:
            return fn(*args, **kwargs)
        except TransientError:
            if attempt == max_retries - 1:
                raise
            time.sleep(delay)
            delay = min(delay * 2, 30)
```

### JavaScript

```javascript
async function call(method, path, body = "") {
  // … 按认证文档构造已签名请求…
  const res = await fetch(url, { method, headers, body: body || undefined });

  if (res.status === 429 || res.status >= 500) {
    throw new TransientError(res.status, await res.text());
  }

  const payload = await res.json();
  if (payload.success === true) return payload.data;

  const codeKey = payload.codeKey || String(payload.code);
  if (res.status === 401) {
    throw new AuthError(codeKey, payload.msg || payload.message);
  }
  throw new BusinessError(codeKey, payload.msg);
}
```

<Tip>
  对于 `429` 与 `5xx`，采用**带抖动的指数退避**：初始 ~1s，每次失败翻倍，封顶 ~30s，并添加少量随机偏移以避免同步重试。
</Tip>
