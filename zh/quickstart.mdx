本指南将带你完成第一次调用 MC OpenAPI 的完整流程：创建 API Key、使用 **HMAC-SHA256 签名 v2** 对请求签名，并通过网关调用汇率接口。

<Note>
  开始前需要一个 MC 平台账号。如果没有，请联系你的 MC 对接人。自 v3.7.0 起必须使用签名 **v2**（每次请求带唯一 `X-Nonce`）。
</Note>

<Steps>
  <Step title="创建 API Key">
    登录 MC 平台，进入 **API Key 管理**，点击 **创建**。为 Key 命名，并勾选需要的权限：

    - `READ` — 只读操作（余额、列表、查询）
    - `TRADE` — 交易操作（下单 / 撤单）
    - `WITHDRAW` — 提币与划转操作

    可选配置 IP 白名单（最多 15 个 IPv4 地址）。立即复制 **Access Key** 和 **Secret Key** —— Secret Key 仅显示一次。

    <Warning>
      切勿将 Secret Key 提交到源代码库，也不要在客户端代码中暴露。
    </Warning>
  </Step>
  <Step title="对请求签名（v2）">
    每个鉴权请求至少需要四个请求头：

    | Header | 说明 |
    | --- | --- |
    | `X-Access-Key` | 你的 Access Key |
    | `X-Timestamp` | 当前 Unix **毫秒** 时间戳（需在服务器时间 ±10 分钟内） |
    | `X-Nonce` | **每次请求唯一**（推荐 UUID v4） |
    | `X-Signature` | 待签名字符串的 HMAC-SHA256 结果的 **Base64** 编码 |

    用 `\n` 拼接以下部分构造待签名字符串，末尾附加一个 `\n`：

    ```text
    {HTTP_METHOD}
    {REQUEST_URI}
    {QUERY_STRING}       # 无 query 时整行省略（不要空行）
    {SIGNED_HEADERS}     # x-access-key&x-nonce&x-timestamp，小写并按 key 排序
    {TIMESTAMP}
    {NONCE}
    [{BODY_SHA256_HEX}]  # 仅 POST/PUT/PATCH 且 body 非空时追加
    ```

    完整规则与 Python / Java / Node.js 示例请参见 [认证](/zh/authentication)。
  </Step>
  <Step title="调用汇率接口">
    `GET /openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT` 仅需 `READ` 权限，适合用来验证签名实现。

    ```bash
    ACCESS_KEY='YOUR_ACCESS_KEY'
    SECRET_KEY='YOUR_SECRET_KEY'
    URI='/openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT'
    TIMESTAMP=$(python3 -c 'import time; print(int(time.time()*1000))')
    NONCE=$(python3 -c 'import uuid; print(uuid.uuid4())')
    
    SIGNATURE=$(ACCESS_KEY="$ACCESS_KEY" SECRET_KEY="$SECRET_KEY" URI="$URI" TIMESTAMP="$TIMESTAMP" NONCE="$NONCE" python3 << 'PY'
    import base64, hashlib, hmac, os
    method = "GET"
    uri = os.environ["URI"]
    access_key = os.environ["ACCESS_KEY"]
    secret_key = os.environ["SECRET_KEY"]
    timestamp = os.environ["TIMESTAMP"]
    nonce = os.environ["NONCE"]
    headers = "&".join(
        f"{k}={v}"
        for k, v in sorted(
            {
                "x-access-key": access_key,
                "x-nonce": nonce,
                "x-timestamp": timestamp,
            }.items()
        )
    )
    string_to_sign = f"{method}\n{uri}\n{headers}\n{timestamp}\n{nonce}\n"
    sig = base64.b64encode(
        hmac.new(secret_key.encode(), string_to_sign.encode(), hashlib.sha256).digest()
    ).decode()
    print(sig)
    PY
    )
    
    curl -s \
      -H "X-Access-Key: ${ACCESS_KEY}" \
      -H "X-Signature: ${SIGNATURE}" \
      -H "X-Timestamp: ${TIMESTAMP}" \
      -H "X-Nonce: ${NONCE}" \
      -H "X-Signature-Version: 2" \
      "https://test-gateway.mcconnects.com${URI}"
    ```

    成功响应：

    ```json
    {
      "status": 200,
      "code": "200",
      "success": true,
      "data": 98500.12
    }
    ```
