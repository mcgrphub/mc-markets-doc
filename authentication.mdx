<Note>
  **Version note (v3.7.0):** Starting with v3.7.0, OpenAPI Category C AK/SK uses **signature v2** (hard cutover; **v1 is not supported**). Every request must include `X-Nonce`. For `POST` / `PUT` / `PATCH` with a non-empty body, the request body SHA-256 must be included in the signature. The v3.4.0 documentation is retained for historical reference only.
</Note>

All OpenAPI requests must be sent to the **API Gateway** and authenticated with an **Access Key (AK)** and **Secret Key (SK)** pair. The gateway validates your signature, enforces nonce replay protection, checks API Key permissions, and forwards the request to the appropriate backend service.

## Base URL

The production gateway is `https://gateway.mcconnects.com`. All paths in this documentation are relative to that host. For example, when the documented path is `/openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT`, the full request URL is:

```text
https://gateway.mcconnects.com/openapi/v1/mc-account/exchange-rate/query/CRYPTO/BTC/USDT
```

<Warning>
  Do not call backend services directly. Always use the gateway URL so that authentication and permission checks are enforced.
</Warning>

## Credentials

### Access Key and Secret Key

Each API Key consists of two parts:

| Credential | Format | Example |
| --- | --- | --- |
| **Access Key (AK)** | `AK` \+ 16 uppercase hex characters | `AKF013EB0CC51E4BEB` |
| **Secret Key (SK)** | `SK` \+ Base64URL-encoded random string | `SKTWRUHyCkE2eOehHp9McVWPjYusIeo8pMun9km06JuHc` |

- The **Secret Key is shown only once** when the API Key is created. Store it securely — it cannot be retrieved again.
- Each API Key is bound to a user account and business line.
- API Keys can be enabled, disabled, or deleted from the user portal.
- You can create up to **30 API Keys** per user.

## Required headers

Every authenticated request must include the following HTTP headers:

| Header | Required | Description |
| --- | --- | --- |
| `X-Access-Key` | Yes | Your Access Key (AK) |
| `X-Signature` | Yes | HMAC-SHA256 signature v2 (Base64-encoded) |
| `X-Timestamp` | Yes | Current Unix timestamp in **milliseconds** |
| `X-Nonce` | Yes | **Unique per request**; UUID v4 recommended, or ≥32 hex characters |
| `Content-Type` | Recommended for POST/PUT/PATCH | Use `application/json` when sending JSON |
| `X-Signature-Version` | No | Suggested value `2` for troubleshooting |

HTTP header names are **case-insensitive** (for example, `x-access-key` and `X-Access-Key` are equivalent). In the string to sign, header names are always **lowercase**.

<Warning>
  Do not send `Authorization: Bearer ...` together with AK/SK headers. If a Bearer token is present, the gateway treats the request as a web session call (Category B mode) and will not use your AK/SK signature.
</Warning>

## Request signing v2

### Overview

Signing prevents tampering and replay attacks. For each request:

1. Generate a unique `X-Nonce`.
2. Build the **string to sign** from the HTTP method, URI, query parameters, signed headers, timestamp, nonce, and (for path B) the request body hash.
3. Compute **HMAC-SHA256** using your Secret Key.
4. Base64-encode the result and send it as the `X-Signature` header.

### String to sign

```text
{HTTP_METHOD}
{REQUEST_URI}
{QUERY_STRING}
{SIGNED_HEADERS}
{TIMESTAMP}
{NONCE}
[{BODY_SHA256_HEX}]
```

| Part | Description |
| --- | --- |
| `HTTP_METHOD` | Uppercase HTTP method, e.g. `GET`, `POST`, `PUT`, `PATCH`, `DELETE` |
| `REQUEST_URI` | Full gateway path without the host, e.g. `/openapi/v1/mc-account/symbol/list` |
| `QUERY_STRING` | Query parameters sorted by key (ASCII ascending), formatted as `key1=value1&key2=value2`. **Omit the entire line** if there are no query parameters (same as v1; do **not** insert a blank line) |
| `SIGNED_HEADERS` | `x-access-key`, `x-nonce`, and `x-timestamp` sorted by key and joined as `key=value&...` |
| `TIMESTAMP` | Same value as `X-Timestamp` (milliseconds) |
| `NONCE` | Same value as `X-Nonce` |
| `BODY_SHA256_HEX` | **Path B only**: see below; lowercase hex, 64 characters |

Append a final newline `\n` at the end.

#### Signed headers

| HTTP header | In string to sign |
| --- | --- |
| `X-Access-Key` | `x-access-key={value}` |
| `X-Nonce` | `x-nonce={value}` |
| `X-Timestamp` | `x-timestamp={value}` |

Other headers (such as `Content-Type`) **do not** participate in the signature.

<Info>
  The URI used in the signature must match the **gateway-visible path** in the API documentation exactly (including service prefixes such as `/openapi/v1/mc-account/`).
</Info>

#### Path A / Path B (request body)

| Path | Condition | `BODY_SHA256_HEX` line |
| --- | --- | --- |
| **Path A** | `GET`; or `POST`/`PUT`/`PATCH` with an **empty body** (e.g. `Content-Length: 0`) | **Not appended** |
| **Path B** | `POST`/`PUT`/`PATCH` with a **non-empty body** | Append lowercase hex of `SHA-256(raw body bytes)` |

- The client must use the **same raw body bytes** for signing and sending (do not rely on the gateway to re-serialize JSON).
- Maximum request body size is **10 MB** (consistent with gateway `gateway.request.max-body-size-bytes`); exceeding the limit returns **413**.

#### Canonicalization rules (v3.7.0)

- **Do not** repeat the same query key (e.g. `?a=1&a=2`); otherwise **401**.
- **Do not** send multiple values for signed headers (e.g. two `X-Nonce` headers); otherwise **401**.
- Each query key contributes only one value in the signature; the query in the URL must match the query in the string to sign.

### Signature algorithm

```text
signature = Base64( HMAC-SHA256( secretKey, stringToSign ) )
```

- Algorithm: **HMAC-SHA256**
- Key encoding: UTF-8 bytes of the Secret Key string
- Output encoding: **Base64** (standard encoding, with padding)

### Timestamp and nonce

- `X-Timestamp` must be a **millisecond** Unix timestamp (13 digits). The gateway accepts **±10 minutes** of server time.
- `X-Nonce` must be **newly generated for every request**; the same nonce cannot be reused within its validity window (replay protection).
- Nonce format: UUID v4, or a hex string of length ≥32.

## Permissions

Each API Key can be configured with one or more permissions. Individual API endpoints declare required permissions; your key must satisfy **all** declared permissions (AND logic).

| Permission | Code | Description |
| --- | --- | --- |
| Read | `READ` | Read-only operations (balances, lists, queries) |
| Trade | `TRADE` | Trading operations (place/cancel orders) |
| Withdraw | `WITHDRAW` | Withdrawal and transfer operations |

If your API Key lacks the required permission for an endpoint, the gateway returns `401 Unauthorized` with the message `没有访问该接口的权限`.

## IP whitelist (optional)

When creating or editing an API Key, you can configure an IP whitelist (up to 15 IPv4 addresses, comma-separated).

- If a whitelist is configured, requests from non-whitelisted IPs are rejected with `401 Unauthorized`.
- If no whitelist is configured, source IP is not restricted (other authentication checks still apply).

The gateway determines the client IP in this order: `X-Forwarded-For` → `X-Real-IP` → direct connection address.

```python Python
import base64
import hashlib
import hmac
import time
import uuid
import requests

ACCESS_KEY = "YOUR_ACCESS_KEY"
SECRET_KEY = "YOUR_SECRET_KEY"
BASE_URL = "https://gateway.mcconnects.com"

def sign_request_v2(
    method: str,
    uri: str,
    query_params: dict | None = None,
    body: bytes | None = None,
) -> dict:
    timestamp = str(int(time.time() * 1000))
    nonce = str(uuid.uuid4())

    parts = [method.upper(), uri]

    if query_params:
        query_string = "&".join(f"{k}={v}" for k, v in sorted(query_params.items()))
        parts.append(query_string)

    signed_headers = "&".join(
        f"{k}={v}"
        for k, v in sorted(
            {
                "x-access-key": ACCESS_KEY,
                "x-nonce": nonce,
                "x-timestamp": timestamp,
            }.items()
        )
    )
    parts.extend([signed_headers, timestamp, nonce])

    body_bytes = body if body is not None else b""
    if body_bytes and method.upper() in {"POST", "PUT", "PATCH"}:
        parts.append(hashlib.sha256(body_bytes).hexdigest())

    string_to_sign = "\n".join(parts) + "\n"
    signature = base64.b64encode(
        hmac.new(SECRET_KEY.encode(), string_to_sign.encode(), hashlib.sha256).digest()
    ).decode()

    return {
        "X-Access-Key": ACCESS_KEY,
        "X-Timestamp": timestamp,
        "X-Nonce": nonce,
        "X-Signature": signature,
        "X-Signature-Version": "2",
    }

# GET example
uri = "/openapi/v1/mc-account/symbol/list"
headers = sign_request_v2("GET", uri)
print(requests.get(f"{BASE_URL}{uri}", headers=headers).status_code)

# POST path B example
uri = "/openapi/v1/mc-account/transaction/account/record/page"
body = b'{"current":1,"size":10}'
headers = sign_request_v2("POST", uri, body=body)
print(
    requests.post(
        f"{BASE_URL}{uri}",
        headers={**headers, "Content-Type": "application/json"},
        data=body,
    ).status_code
)
```

```java Java
import javax.crypto.Mac;
import javax.crypto.spec.SecretKeySpec;
import java.nio.charset.StandardCharsets;
import java.security.MessageDigest;
import java.util.Base64;
import java.util.Map;
import java.util.TreeMap;
import java.util.UUID;
import java.util.stream.Collectors;

public class McOpenApiSignerV2 {

    public static String sign(String method, String uri,
                              Map<String, String> queryParams,
                              byte[] bodyBytes,
                              String accessKey, String secretKey) throws Exception {
        String timestamp = String.valueOf(System.currentTimeMillis());
        String nonce = UUID.randomUUID().toString();

        StringBuilder sb = new StringBuilder();
        sb.append(method.toUpperCase()).append("\n").append(uri).append("\n");

        if (queryParams != null && !queryParams.isEmpty()) {
            String query = new TreeMap<>(queryParams).entrySet().stream()
                    .map(e -> e.getKey() + "=" + e.getValue())
                    .collect(Collectors.joining("&"));
            sb.append(query).append("\n");
        }

        TreeMap<String, String> signHeaders = new TreeMap<>();
        signHeaders.put("x-access-key", accessKey);
        signHeaders.put("x-nonce", nonce);
        signHeaders.put("x-timestamp", timestamp);
        sb.append(signHeaders.entrySet().stream()
                .map(e -> e.getKey() + "=" + e.getValue())
                .collect(Collectors.joining("&"))).append("\n");
        sb.append(timestamp).append("\n").append(nonce).append("\n");

        if (bodyBytes != null && bodyBytes.length > 0
                && isWriteMethod(method)) {
            sb.append(sha256Hex(bodyBytes)).append("\n");
        }

        Mac mac = Mac.getInstance("HmacSHA256");
        mac.init(new SecretKeySpec(secretKey.getBytes(StandardCharsets.UTF_8), "HmacSHA256"));
        return Base64.getEncoder().encodeToString(
                mac.doFinal(sb.toString().getBytes(StandardCharsets.UTF_8)));
    }

    private static boolean isWriteMethod(String method) {
        String m = method.toUpperCase();
        return "POST".equals(m) || "PUT".equals(m) || "PATCH".equals(m);
    }

    private static String sha256Hex(byte[] bytes) throws Exception {
        byte[] hash = MessageDigest.getInstance("SHA-256").digest(bytes);
        StringBuilder hex = new StringBuilder(hash.length * 2);
        for (byte b : hash) {
            hex.append(String.format("%02x", b));
        }
        return hex.toString();
    }
}
```

```javascript Node.js
const crypto = require("crypto");

function signRequestV2(method, uri, queryParams, bodyBuffer, accessKey, secretKey) {
  const timestamp = String(Date.now());
  const nonce = crypto.randomUUID();
  const parts = [method.toUpperCase(), uri];

  if (queryParams && Object.keys(queryParams).length > 0) {
    const query = Object.keys(queryParams)
      .sort()
      .map((k) => `${k}=${queryParams[k]}`)
      .join("&");
    parts.push(query);
  }

  const signHeaders = {
    "x-access-key": accessKey,
    "x-nonce": nonce,
    "x-timestamp": timestamp,
  };
  const headerStr = Object.keys(signHeaders)
    .sort()
    .map((k) => `${k}=${signHeaders[k]}`)
    .join("&");
  parts.push(headerStr, timestamp, nonce);

  const body = bodyBuffer || Buffer.alloc(0);
  if (body.length > 0 && ["POST", "PUT", "PATCH"].includes(method.toUpperCase())) {
    parts.push(crypto.createHash("sha256").update(body).digest("hex"));
  }

  const stringToSign = parts.join("\n") + "\n";
  const signature = crypto
    .createHmac("sha256", secretKey)
    .update(stringToSign)
    .digest("base64");

  return {
    "X-Access-Key": accessKey,
    "X-Timestamp": timestamp,
    "X-Nonce": nonce,
    "X-Signature": signature,
    "X-Signature-Version": "2",
  };
}
```

## Error responses

Authentication failures typically return HTTP **401 Unauthorized**; request body size violations return **413 Payload Too Large**. The response body is JSON:

```json
{
  "status": 401,
  "code": 401,
  "message": "签名无效"
}
```

| Message | Cause |
| --- | --- |
| `缺少有效鉴权凭证` | No valid AK/SK authentication headers provided |
| `缺少AccessKey, 请在请求头中添加X-Access-Key` | Missing `X-Access-Key` |
| `缺少签名, 请在请求头中添加X-Signature` | Missing `X-Signature` |
| `缺少时间戳, 请在请求头中添加X-Timestamp` | Missing `X-Timestamp` |
| `缺少 nonce, 请在请求头中添加 X-Nonce` | Missing `X-Nonce` |
| `非法 nonce 格式` | Nonce does not match UUID v4 or hex rules |
| `请求重放` | Nonce has already been used |
| `非法重复查询参数: {key}` | Duplicate query key |
| `非法重复请求头: {name}` | Signed header has multiple values |
| `时间戳无效或已过期` | Timestamp outside ±10 minutes |
| `[X-Access-Key]AccessKey无效` | Access Key is invalid |
| `签名无效` | Signature mismatch (including body tampering) |
| `请求体超过限制` | Body exceeds 10 MB (**413**) |
| `没有访问权限, ip: {ip}` | IP not in whitelist |
| `没有访问该接口的权限` | Missing endpoint permission |
| `API Key 未绑定业务线或用户` | API Key configuration error |

## Security best practices

<Tip>
  1. **Never expose your Secret Key** in client-side code, public repositories, or logs.
  2. **Use a new `X-Nonce` on every request** — do not reuse nonces.
  3. **Configure an IP whitelist in production**.
  4. **Grant minimum permissions**.
  5. **Rotate keys regularly**.
  6. **Keep your system clock synchronized** (NTP).
  7. **Use HTTPS only**.
  8. **Upgrade clients to v2 promptly after the gateway deploys v2**; v1 signatures will no longer be accepted.
</Tip>

## Quick checklist

- [ ] Upgraded to **signature v2** with a **unique** `X-Nonce` on every request
- [ ] Request URL uses the **gateway** base URL
- [ ] URI matches the documented gateway path
- [ ] Query matches the string to sign; **no duplicate query keys**
- [ ] Non-empty POST/PUT/PATCH body includes **SHA-256 hex** as the final line
- [ ] Signed body bytes **match** sent body bytes
- [ ] `X-Timestamp` is within ±10 minutes
- [ ] API Key has required permissions for the target endpoint
- [ ] IP is whitelisted (if configured)
