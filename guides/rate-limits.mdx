The Open API Platform enforces rate limits per API key to ensure fair use and reliable performance for all users. When you exceed your plan's limit, the API returns a `429 Too Many Requests` error until your window resets. Building rate-limit-aware retry logic into your integration keeps your application resilient under load.

## Rate limit tiers

Limits are measured in a rolling 60-second window for per-minute quotas, and a rolling 24-hour window for daily quotas.

| Plan | Requests/minute | Requests/day |
|---|---|---|
| Free | 60 | 5,000 |
| Pro | 300 | 50,000 |
| Enterprise | 1,000 | Unlimited |

<Warning>
  Repeatedly exceeding your rate limit in a short period may result in a temporary block on your API key. If you consistently hit your plan's ceiling, [contact support](mailto:support@example.com) to discuss a limit increase or an upgrade to a higher tier.
</Warning>

## Rate limit headers

Every API response includes headers that tell you your current rate limit status. Read these headers proactively to throttle your own requests before hitting a `429`.

| Header | Description |
|---|---|
| `X-RateLimit-Limit` | Maximum number of requests allowed in the current window |
| `X-RateLimit-Remaining` | Number of requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp (UTC) when the current window resets |
| `Retry-After` | Seconds to wait before retrying after a `429` response |

An example response header set might look like this:

```
X-RateLimit-Limit: 300
X-RateLimit-Remaining: 42
X-RateLimit-Reset: 1715342460
Retry-After: 17
```

## Handling 429 errors

When you receive a `429` response, read the `Retry-After` header and wait the indicated number of seconds before retrying. If `Retry-After` is absent, use truncated exponential backoff with jitter to avoid thundering-herd problems.

<CodeGroup>

```python Python
import time
import random
import requests

API_KEY = "YOUR_API_KEY"
BASE_URL = "https://api.example.com/v1"

def request_with_backoff(method: str, path: str, **kwargs) -> requests.Response:
    max_retries = 5
    base_delay = 1.0  # seconds

    for attempt in range(max_retries):
        response = requests.request(
            method,
            f"{BASE_URL}{path}",
            headers={"Authorization": f"Bearer {API_KEY}"},
            **kwargs,
        )

        if response.status_code != 429:
            return response

        retry_after = response.headers.get("Retry-After")
        if retry_after:
            wait = float(retry_after)
        else:
            # Exponential backoff with full jitter
            wait = base_delay * (2 ** attempt) + random.uniform(0, 1)

        print(f"Rate limited. Retrying in {wait:.1f}s (attempt {attempt + 1}/{max_retries})...")
        time.sleep(wait)

    response.raise_for_status()
    return response


# Usage
resources = request_with_backoff("GET", "/resources").json()
```

```javascript JavaScript
const BASE_URL = "https://api.example.com/v1";
const API_KEY = "YOUR_API_KEY";

async function sleep(ms) {
  return new Promise((resolve) => setTimeout(resolve, ms));
}

async function requestWithBackoff(method, path, options = {}) {
  const maxRetries = 5;
  const baseDelay = 1000; // milliseconds

  for (let attempt = 0; attempt < maxRetries; attempt++) {
    const response = await fetch(`${BASE_URL}${path}`, {
      method,
      headers: {
        Authorization: `Bearer ${API_KEY}`,
        "Content-Type": "application/json",
        ...options.headers,
      },
      ...options,
    });

    if (response.status !== 429) {
      return response;
    }

    const retryAfter = response.headers.get("Retry-After");
    let wait;
    if (retryAfter) {
      wait = parseFloat(retryAfter) * 1000;
    } else {
      // Exponential backoff with full jitter
      wait = baseDelay * Math.pow(2, attempt) + Math.random() * 1000;
    }

    console.log(
      `Rate limited. Retrying in ${(wait / 1000).toFixed(1)}s (attempt ${attempt + 1}/${maxRetries})...`
    );
    await sleep(wait);
  }

  throw new Error(`Request failed after ${maxRetries} retries`);
}

// Usage
const response = await requestWithBackoff("GET", "/resources");
const resources = await response.json();
```

</CodeGroup>

<Tip>
  Use batch endpoints wherever the API offers them. Fetching 100 resources in a single `POST /v1/resources/batch` request consumes only one request against your quota instead of 100 individual `GET` calls — dramatically reducing your exposure to rate limits.
</Tip>
