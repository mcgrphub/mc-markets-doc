This guide walks you through a full signed request against the MC OpenAPI gateway. By the end you will have a working call to the `GET /openapi/v1/exchange-rate/query/usd` endpoint that returns live USD conversion rates.

## Prerequisites

Before you start, make sure you have:

- An MC account with API access enabled.
- An **Access Key** and **Secret Key** — see [Authentication](/authentication#creating-api-keys) for how to create them.
- The `READ` permission enabled on your key (required for the exchange-rate endpoint).
- A tool that can send HTTPS requests: `curl`, Python 3.8\+, or Node.js 18\+.

<Warning>
  Your Secret Key is shown only once at creation time. Store it in a secrets manager or environment variable — never commit it to source control.
</Warning>

## Step 1 — Choose an environment

| Environment | Base URL |
| --- | --- |
| Test | `https://test-gateway.mcconnects.com` |
| Production | `https://gateway.mcconnects.com` |

The rest of this guide uses production. Swap the host to point at the test gateway.

## Step 2 — Understand what you are signing

Every MC OpenAPI request is signed with HMAC-SHA256 over a canonical string:

```text
{METHOD}\n{PATH}\n{SORTED_QUERY_STRING}\n{BODY_OR_EMPTY}\n{TIMESTAMP}
```

For a `GET /openapi/v1/exchange-rate/query/usd` call with no query parameters and no body, the canonical string is:

```text
GET
/openapi/v1/exchange-rate/query/usd


1731657600000
```

Note the blank lines for the empty query string and empty body. Full details are in the [Authentication](/authentication#signing-a-request) reference.

## Step 3 — Send the request

### cURL

```bash
ACCESS_KEY="your_access_key"
SECRET_KEY="your_secret_key"
TS=$(date +%s000)
METHOD="GET"
PATH_="/openapi/v1/exchange-rate/query/usd"

CANONICAL=$(printf "%s\n%s\n\n\n%s" "$METHOD" "$PATH_" "$TS")
SIGNATURE=$(printf "%s" "$CANONICAL" | openssl dgst -sha256 -hmac "$SECRET_KEY" -hex | awk '{print $2}')

curl --request GET \
  --url "https://gateway.mcconnects.com${PATH_}" \
  --header "X-Access-Key: ${ACCESS_KEY}" \
  --header "X-Signature: ${SIGNATURE}" \
  --header "X-Timestamp: ${TS}" \
  --header "Accept: application/json"
```

### Python

```python
import hmac, hashlib, time, requests, os

BASE_URL = "https://gateway.mcconnects.com"
ACCESS_KEY = os.environ["MC_ACCESS_KEY"]
SECRET_KEY = os.environ["MC_SECRET_KEY"]

method = "GET"
path = "/openapi/v1/exchange-rate/query/usd"
query = ""
body = ""
ts = str(int(time.time() * 1000))

canonical = f"{method}\n{path}\n{query}\n{body}\n{ts}"
signature = hmac.new(SECRET_KEY.encode(), canonical.encode(), hashlib.sha256).hexdigest()

resp = requests.get(
    BASE_URL + path,
    headers={
        "X-Access-Key": ACCESS_KEY,
        "X-Signature": signature,
        "X-Timestamp": ts,
        "Accept": "application/json",
    },
)
print(resp.status_code, resp.json())
```

### Node.js

```javascript
import crypto from "crypto";

const BASE_URL = "https://gateway.mcconnects.com";
const ACCESS_KEY = process.env.MC_ACCESS_KEY;
const SECRET_KEY = process.env.MC_SECRET_KEY;

const method = "GET";
const path = "/openapi/v1/exchange-rate/query/usd";
const query = "";
const body = "";
const ts = Date.now().toString();

const canonical = `${method}\n${path}\n${query}\n${body}\n${ts}`;
const signature = crypto.createHmac("sha256", SECRET_KEY).update(canonical).digest("hex");

const res = await fetch(BASE_URL + path, {
  method,
  headers: {
    "X-Access-Key": ACCESS_KEY,
    "X-Signature": signature,
    "X-Timestamp": ts,
    Accept: "application/json",
  },
});

console.log(res.status, await res.json());
```

## Step 4 — Read the response

A successful response looks like this:

```json
{
  "code": "0",
  "msg": "success",
  "success": true,
  "data": [
    { "symbolType": "FIAT", "baseCurrency": "USD", "targetCurrency": "EUR", "rate": "0.9187" },
    { "symbolType": "FIAT", "baseCurrency": "USD", "targetCurrency": "JPY", "rate": "155.42" },
    { "symbolType": "CRYPTO", "baseCurrency": "USD", "targetCurrency": "BTC", "rate": "0.0000148" }
  ]
}
```

- `code == "0"` — the call succeeded.
- `data` — the payload (see the [Exchange Rate endpoint](/api-reference/endpoints/exchange-rate) for the full schema).

If `code` is not `"0"`, inspect `msg` and consult the [Error Handling](/concepts/errors) reference.
