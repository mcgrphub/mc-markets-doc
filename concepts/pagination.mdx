MC OpenAPI uses page-number pagination on every list endpoint. You request a page with `pageNum` and `pageSize`, and the response returns the current page of items plus enough metadata to iterate to the next page.

## Request parameters

Paginated endpoints accept the following parameters (in the query string for `GET`, in the JSON body for `POST`, depending on the endpoint — see the individual endpoint pages):

<ParamField query="pageNum" default="1" type="integer">
  The page number to retrieve. Pages are 1-indexed.
</ParamField>

<ParamField query="pageSize" default="20" type="integer">
  Number of items to return per page. Typical maximum is `100`; check the specific endpoint for its own limit.
</ParamField>

Most list endpoints accept additional filter parameters (for example, `startTime`, `endTime`, `symbol`, `type`). See the [API Reference](/api-reference/overview) for the exact parameters supported by each endpoint.

## Response shape

A paginated response places a page object inside `data`:

```json
{
  "code": "0",
  "msg": "success",
  "success": true,
  "data": {
    "pageNum": 1,
    "pageSize": 20,
    "total": 132,
    "pages": 7,
    "list": [
      {
        "transactionId": "T-1001",
        "amount": "125.50",
        "currency": "USDT",
        "createdTime": 1731657600000
      }
    ]
  }
}
```

<ResponseField name="data.pageNum" type="integer">
  The current page number (1-indexed).
</ResponseField>

<ResponseField name="data.pageSize" type="integer">
  Number of items requested per page.
</ResponseField>

<ResponseField name="data.total" type="integer">
  Total number of items across all pages matching your query.
</ResponseField>

<ResponseField name="data.pages" type="integer">
  Total number of pages available given `pageSize` and `total`.
</ResponseField>

<ResponseField name="data.list" type="array">
  The items on the current page. Item shape depends on the endpoint.
</ResponseField>

## Iterating all pages

Loop while `pageNum < pages`, incrementing `pageNum` on each request. The example below fetches every transaction record for a given account.

### Python

```python
import time, hmac, hashlib, requests

BASE_URL = "https://gateway.mcconnects.com"
ACCESS_KEY = "YOUR_ACCESS_KEY"
SECRET_KEY = "YOUR_SECRET_KEY"

def sign(method, path, query, body, ts):
    canonical = f"{method}\n{path}\n{query}\n{body}\n{ts}"
    return hmac.new(SECRET_KEY.encode(), canonical.encode(), hashlib.sha256).hexdigest()

def fetch_all_transactions(page_size=100):
    path = "/openapi/v1/transaction/account/record/page"
    page_num = 1
    all_items = []

    while True:
        body = f'{{"pageNum":{page_num},"pageSize":{page_size}}}'
        ts = str(int(time.time() * 1000))
        signature = sign("POST", path, "", body, ts)
        headers = {
            "X-Access-Key": ACCESS_KEY,
            "X-Signature": signature,
            "X-Timestamp": ts,
            "Content-Type": "application/json",
        }
        r = requests.post(BASE_URL + path, headers=headers, data=body)
        r.raise_for_status()
        page = r.json()["data"]
        all_items.extend(page["list"])

        if page_num >= page["pages"]:
            break
        page_num += 1

    return all_items
```

### JavaScript (Node.js)

```javascript
import crypto from "crypto";

const BASE_URL = "https://gateway.mcconnects.com";
const ACCESS_KEY = process.env.MC_ACCESS_KEY;
const SECRET_KEY = process.env.MC_SECRET_KEY;

function sign(method, path, query, body, ts) {
  const canonical = `${method}\n${path}\n${query}\n${body}\n${ts}`;
  return crypto.createHmac("sha256", SECRET_KEY).update(canonical).digest("hex");
}

async function fetchAllTransactions(pageSize = 100) {
  const path = "/openapi/v1/transaction/account/record/page";
  let pageNum = 1;
  const all = [];

  while (true) {
    const body = JSON.stringify({ pageNum, pageSize });
    const ts = Date.now().toString();
    const signature = sign("POST", path, "", body, ts);

    const res = await fetch(BASE_URL + path, {
      method: "POST",
      headers: {
        "X-Access-Key": ACCESS_KEY,
        "X-Signature": signature,
        "X-Timestamp": ts,
        "Content-Type": "application/json",
      },
      body,
    });

    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const { data } = await res.json();
    all.push(...data.list);
    if (pageNum >= data.pages) break;
    pageNum += 1;
  }
  return all;
}
```

<Tip>
  Use the largest `pageSize` your endpoint allows to minimize round-trips and reduce the chance of hitting [rate limits](/guides/rate-limits).
</Tip>
