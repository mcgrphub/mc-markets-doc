## Environments

| Environment | Base URL |
| --- | --- |
| Test | `https://test-gateway.mcconnects.com` |
| Production | `https://gateway.mcconnects.com` |

All endpoints share the path prefix `/openapi/v1/`. See [Base URL](/api-reference/base-url) for details.

## Endpoint groups

MC OpenAPI is organized into six functional groups. Every endpoint requires a signed request; some also require additional permissions on the Access Key.

| Group | Purpose | Required scope |
| --- | --- | --- |
| [Trading](/api-reference/endpoints/trading) | Transaction records by ID or paginated queries | `READ` |
| [Finance Account](/api-reference/endpoints/finance-account) | Deposits, withdrawals, fixed-term products, portfolio overview | `READ`, `TRADE`, `WITHDRAW` |
| [Currency](/api-reference/endpoints/currency) | Symbol / exchange metadata | `READ` |
| [Fund Account](/api-reference/endpoints/fund-account) | Spot positions and proof-of-reserves snapshots | `READ` |
| [Funding Fee](/api-reference/endpoints/funding-fee) | Perpetual funding-fee history | `READ` |
| [Exchange Rate](/api-reference/endpoints/exchange-rate) | FX and crypto conversion rates | `READ` |

## Request lifecycle

Every call follows the same lifecycle:

1. **Build** the request — HTTP method, path (`/openapi/v1/...`), query string, and optional JSON body.
2. **Sign** the request — compute an HMAC-SHA256 signature over the canonical string using your Secret Key.
3. **Send** the request over HTTPS with the `X-Access-Key`, `X-Signature`, and `X-Timestamp` headers.
4. **Handle** the response — inspect the HTTP status, then the `code` field in the JSON envelope.

<Note>
  Timestamps must be within a small window of the gateway clock. Always use the current UTC time in milliseconds when signing.
</Note>
