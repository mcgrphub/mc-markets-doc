## Environments

| Environment | Base URL |
| --- | --- |
| Test | `https://test-gateway.mcconnects.com` |
| Production | `https://gateway.mcconnects.com` |

All endpoints share the path prefix `/openapi/v1/`. See [Base URL](/api-reference/base-url) for details.

## Endpoint groups

MC OpenAPI endpoints are generated from `openapi-exposed` Specs and grouped by gateway service in the API Reference sidebar.

| Group | Purpose | Required scope (typical) |
| --- | --- | --- |
| Account (`/openapi/v1/mc-account/`) | Trading records, finance, MLP, currency, fund account, exchange rate | `READ`, `TRADE`, `WITHDRAW` |
| Trade (`/openapi/v1/mc-trade/`) | Orders, positions, deals, trade account | `READ`, `TRADE` |
| Risk (`/openapi/v1/mc-risk/`) | Funding rate, trade symbols | `READ` |
| Aggregator (`/openapi/v1/mc-aggregator/`) | K-line and symbol ranking | `READ` |

See the [API Reference overview](/api-reference/overview) for Spec file mapping.

## Request lifecycle

Every call follows the same lifecycle:

1. **Build** the request — HTTP method, path (`/openapi/v1/...`), query string, and optional JSON body.
2. **Sign** the request — compute signature **v2** (unique `X-Nonce`; include body SHA-256 for non-empty write bodies) with HMAC-SHA256 (**Base64**).
3. **Send** the request over HTTPS with the `X-Access-Key`, `X-Signature`, `X-Timestamp`, and `X-Nonce` headers.
4. **Handle** the response — inspect the HTTP status, then the `code` field in the JSON envelope.

<Note>
  Timestamps must be within a small window of the gateway clock. Always use the current UTC time in milliseconds when signing.
</Note>
