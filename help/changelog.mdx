<Update label="2026-07-21" description="API Reference redesign">
  - API Reference sidebar is grouped by **Market data** / **Trading** / **Finance** (not by Spec services Account / Trade / Risk / Aggregator).
  - Reader-facing name unified to **Vault account**; removed the retired App global search endpoint.
  - API playground disabled (custom AK/SK signing required).
</Update>

<Update label="2026-07-20" description="OpenAPI surface expansion">
  - **Signing**: Updated [Authentication](/authentication) and [Quickstart](/quickstart). Every request requires `X-Nonce`; non-empty `POST`/`PUT`/`PATCH` bodies must include SHA-256 in the string to sign; `X-Signature` is Base64. Corrected outdated signing notes in Requests & Responses, FAQ, and Troubleshooting.
  - **Finance Account**: added product catalog, current positions, and redemption history; demand/fixed-term redemption permission is `TRADE` (not `WITHDRAW`).
  - **Trading**: added flash conversion `POST /openapi/v1/transaction/exchange/v2` (`TRADE`).
  - **Deal Orders**: opened paginated deal list and deal detail.
  - **Vault account**: opened deposit/withdraw, assets, charts, and config queries.
  - **Fund Account**: added spot positions by `spotId`.
  - **Funding Rate / Trade Symbols** (mc-risk): opened funding-rate history pagination and trade symbol list.
  - **Market Data** (mc-aggregator-service): opened K-line and symbol ranking.
</Update>

<Update label="2026-07-16" description="Documentation refresh">
  - Published a full documentation site for MC OpenAPI covering all six endpoint groups: Trading, Finance Account, Currency, Fund Account, Funding Fee, and Exchange Rate.
  - Added [Authentication](/authentication) reference with HMAC-SHA256 canonical-string spec and copy-paste examples in Python, Java, and Node.js.
  - Added [Rate Limits](/guides/rate-limits) with exponential-backoff helpers.
  - Added [Polling for Updates](/guides/webhooks) — MC OpenAPI is request-response only; use these patterns to detect new activity.
  - Added ready-to-use client [code samples](/guides/sdks) in Python, Node.js, Java, and Go.
</Update>

<Update label="v2 endpoints" description="Recommended over v1">
  These v2 endpoints supersede their v1 counterparts. Existing v1 endpoints continue to work; migrate at your own pace.

  - `GET /openapi/v1/mc-account/fund/account/spot/positions/v2` — spot positions with expanded fields.
  - `POST /openapi/v1/mc-account/funding-fee-record/page/v2` — funding-fee history grouped by `positionId` and `dealTime`.

  See [Versioning](/api-reference/versioning) for the full v2 list.
</Update>

<Update label="Initial API surface" description="MC OpenAPI v0.0.1">
  MC OpenAPI is generally available with the following endpoint groups:

  - **Trading** — transaction records by ID and paginated queries.
  - **Finance Account** — deposit / withdraw / fixed-term product subscriptions plus portfolio overview.
  - **Currency** — symbol networks, exchangeable symbols, per-symbol details.
  - **Fund Account** — spot positions and Proof-of-Reserves (PoRv2) snapshots.
  - **Funding Fee** — perpetual funding-fee history.
  - **Exchange Rate** — pair, USD, and flash-conversion rate queries.

  All endpoints share the path prefix `/openapi/v1/` and are secured with AK/SK \+ HMAC-SHA256 signing.
</Update>
