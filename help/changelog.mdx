### v1.3.0 â€” January 2025

<Badge>New Features</Badge>

- **Cursor-based pagination on all list endpoints.** All `GET` endpoints that return collections now support cursor-based pagination via the `cursor` query parameter, in addition to the existing offset-based approach. Cursor pagination is more efficient for large datasets and prevents duplicate or skipped records when data changes between pages.

- **`X-Request-ID` header on every response.** Every API response now includes an `X-Request-ID` header containing a unique identifier for that request. Reference this value when contacting support to help the team trace your request in server logs instantly.

- **New `user.updated` webhook event.** You can now subscribe to `user.updated` events to receive real-time notifications whenever a user record is modified. Register the event from **Settings â†’ Webhooks** in your dashboard.

### v1.2.0 â€” November 2024

<Badge>New Features</Badge>

- **Webhook signature verification via HMAC-SHA256.** Every webhook delivery now includes an `X-Webhook-Signature` header containing an HMAC-SHA256 signature of the request body, signed with your webhook secret. Verify this signature in your endpoint handler to confirm that events are genuinely sent by the Open API Platform and haven't been tampered with.

- **`per_page` limit increased to 100.** The `per_page` query parameter on list endpoints now accepts values up to `100`, raised from the previous maximum of `50`. This reduces the number of requests needed to paginate through large collections.

- **Filter `GET /v1/resources` by `status`.** The resource listing endpoint now accepts a `status` query parameter so you can retrieve only the records matching a specific status value (e.g. `active`, `inactive`, `pending`) without filtering client-side.

  ```http
  GET /v1/resources?status=active
  ```

### v1.1.0 â€” September 2024

<Badge>New Features</Badge>

- **Initial webhook support.** You can now receive real-time event notifications via webhooks. This release introduces support for two events: `resource.created` and `resource.deleted`. Register your endpoint URL from **Settings â†’ Webhooks**. See the [Webhooks guide](/guides/webhooks) for setup instructions.

- **Rate limit headers on all responses.** Every API response now includes the following headers so you can monitor your consumption and back off proactively:

  | Header | Description |
  |---|---|
  | `X-RateLimit-Limit` | The maximum number of requests allowed in the current window. |
  | `X-RateLimit-Remaining` | The number of requests remaining in the current window. |
  | `X-RateLimit-Reset` | The UTC epoch timestamp at which the current window resets. |

- **`error.details` array in validation error responses.** `422 Unprocessable Entity` responses now include an `error.details` array that lists every field-level validation failure in a single response, so you can fix all issues in one round trip rather than discovering them one at a time.
