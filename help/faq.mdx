## Getting started

<AccordionGroup>
  <Accordion title="How do I get an Access Key and Secret Key?">
    Sign in to your MC account and open **API Management**. Create a new key, choose the permissions you need (`READ`, `TRADE`, `WITHDRAW`), and copy both the Access Key and the Secret Key immediately — the Secret Key is shown only once. Full walkthrough: [Authentication](/authentication#creating-api-keys).
  </Accordion>

  <Accordion title="What is the difference between the test and production gateways?">
    - **Test:** `https://test-gateway.mcconnects.com` — sandbox for building integrations. Uses test balances and does not affect real funds.
    - **Production:** `https://gateway.mcconnects.com` — live traffic. Real balances, real money movement.

    Keys are environment-specific: a test key will not work against production and vice versa. See [Base URL](/api-reference/base-url).
  </Accordion>

  <Accordion title="Which permission does an endpoint require?">
    - `READ` — most query endpoints (transactions, positions, exchange rates, symbol metadata, PoR snapshots).
    - `TRADE` — finance-account deposit and fixed-term subscriptions.
    - `WITHDRAW` — finance-account withdrawals.

    The required scope is listed on each endpoint page under [API Reference](/api-reference/overview).
  </Accordion>
</AccordionGroup>

## Authentication

<AccordionGroup>
  <Accordion title="What exactly do I sign?">
    From v3.7.0, use signature **v2**. The string to sign includes method, URI, optional query, signed headers (`x-access-key` / `x-nonce` / `x-timestamp`), timestamp, nonce, and (for non-empty `POST`/`PUT`/`PATCH` bodies) the lowercase SHA-256 hex of the raw body. Put the HMAC-SHA256 digest in `X-Signature` as **Base64** (not hex). Every request needs a unique `X-Nonce`. Full rules: [Authentication](/authentication).
  </Accordion>

  <Accordion title="I get an invalid signature. What's wrong?">
    In order of likelihood:

    1. Still using the v1 string to sign (missing `X-Nonce` / body hash).
    2. The string to sign does not match what the gateway expects — print it and compare byte-for-byte.
    3. You signed different body bytes than you sent (extra whitespace, reordered JSON). Serialize once and reuse the exact bytes.
    4. You used the Access Key in place of the Secret Key.
    5. Query parameters are not sorted by key ascending, or a query key is repeated.

    See also [Troubleshooting](/help/troubleshooting).
  </Accordion>

  <Accordion title="I get code 40002 (timestamp expired). What's wrong?">
    Your machine's clock is skewed from the gateway. `X-Timestamp` must be current UTC time in **milliseconds** (not seconds), and within a small window of the server clock. Sync via NTP.
  </Accordion>

  <Accordion title="Can I rotate keys without downtime?">
    Yes. Create a new key with the same permissions, deploy it, then disable the old key in the console. Both keys are valid concurrently until you disable the old one.
  </Accordion>
</AccordionGroup>

## Requests and pagination

<AccordionGroup>
  <Accordion title="Which endpoints return a paginated response?">
    Every list-style endpoint (transaction records, funding-fee history) returns `pageNum`, `pageSize`, `total`, `pages`, and `list`. See [Pagination](/concepts/pagination).
  </Accordion>

  <Accordion title="What is the largest pageSize I can request?">
    Most list endpoints allow up to `100`. Check the specific endpoint under [API Reference](/api-reference/overview) for its documented limit.
  </Accordion>

  <Accordion title="How do I get real-time updates?">
    MC OpenAPI does not currently deliver webhooks or push events. Use the [polling patterns](/guides/webhooks) for detecting new activity efficiently.
  </Accordion>
</AccordionGroup>

## Data model

<AccordionGroup>
  <Accordion title="What is Proof of Reserves (PoRv2)?">
    A per-user Merkle-tree snapshot of assets held on the platform. Call `GET /openapi/v1/mc-account/fund/account/por` to fetch your snapshot and audit path. See the fund-account operations under **Account** in the [API Reference](/api-reference/overview) for the full response schema.
  </Accordion>

  <Accordion title="What is the difference between v1 and v2 endpoints?">
    Some resources (spot positions, funding-fee records) have both v1 and v2 variants. The v2 versions are the current recommendation and return additional fields. See [Versioning](/api-reference/versioning) for the full list.
  </Accordion>

  <Accordion title="Are numeric fields returned as numbers or strings?">
    All monetary amounts, prices, and rates are returned as **strings** to preserve precision. Parse them with a decimal library (Python `Decimal`, JavaScript `bignumber.js`, Java `BigDecimal`) — never with floating-point.
  </Accordion>
</AccordionGroup>

## Operational

<AccordionGroup>
  <Accordion title="Are there official SDKs?">
    Not yet. Copy-paste-ready client snippets for Python, Node.js, Java, and Go are on [Code Samples](/guides/sdks).
  </Accordion>

  <Accordion title="How are rate limits enforced?">
    Per-Access-Key at the gateway. Exceeding the limit returns HTTP `429` with `code: "42901"`. See [Rate Limits](/guides/rate-limits) for the recommended backoff strategy.
  </Accordion>

  <Accordion title="Where do I report a bug or request a feature?">
    Contact MC support with a description of the issue, the affected endpoint, the timestamp of a failing request, and — if possible — the response body (with Secret Key redacted).
  </Accordion>
</AccordionGroup>
